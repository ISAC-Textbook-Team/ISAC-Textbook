上一节主要从概念与演进逻辑的角度回答了“ISAC 是什么”以及“ISAC 如何逐步发展而来”的问题。本节进一步讨论一个更为根本的问题：\emph{ISAC 的核心价值究竟是什么，以及为什么这种价值并不是传统独立系统简单叠加所能够替代的？} 我们将从频谱效率、硬件成本、部署效率和业务闭环能力等几个维度展开分析，揭示 ISAC 相较于传统独立通信系统和独立感知系统所具有的系统级优势。更重要的是，本节将说明，这些优势并不是彼此孤立的局部收益，而是共同指向一种网络能力形态的根本变化：无线网络不再只是承担“传输信息”的单一角色，而开始具备“感知环境、理解状态、支撑决策”的综合服务能力\cite{Liu2022ISAC,Gonzalez2024integrated,Saad2020Vision6G}。正是在这一意义上，ISAC 不应被理解为一种可有可无的功能附加，而应被理解为未来 6G 网络的重要内生能力之一。

\subsection{Why is ISAC not merely an "optional addition to existing functions"?}
\label{subsec:ch1-isac-core-value-why}
\subsubsection{The value logic from independent systems to integrated systems}
\label{subsubsec:ch1-isac-core-value-logic}

在过去数十年中，通信系统与感知系统分别沿着各自清晰而独立的技术路径发展，逐步形成了完整且成熟的理论体系与工程体系。通信系统的核心任务始终是实现信息在不同节点之间的可靠、高效传输，其设计重点主要围绕信道容量、频谱效率、传输可靠性、覆盖能力和时延等指标展开\cite{Goldsmith2005}. 无论是从 1G 到 5G 的代际演进，还是从 FDMA、TDMA 到 OFDMA、Massive MIMO 的关键技术发展，通信系统的根本目标都可以归结为“如何更高效地传送比特”。从这一意义上说，传统通信系统对外部物理世界的关注是间接的：它关心的是信号传播环境如何影响链路质量，而并不直接关心环境本身的语义、结构与动态状态。
与之相对，独立感知系统，尤其是雷达、定位和环境探测系统，则以“理解物理世界”为首要任务，其设计目标集中于目标检测、参数估计、状态跟踪和场景感知等能力\cite{SkolnikRadarBook}. 这类系统关心的是目标在哪里、以何种速度运动、环境中存在哪些散射体或障碍物，以及场景状态如何随时间变化。然而，传统感知系统通常并不直接负责将这些信息以网络化方式传递给远端节点，也不直接负责与上层通信、计算和控制流程进行一体化协同。换言之，在长期的技术演进中，通信系统与感知系统分别承担了“传信息”和“看环境”这两类彼此相关、但工程上相对分离的任务。
这种分工在较长时期内是合理且高效的。原因在于，早期网络业务大多可以被划分为相对独立的功能模块：通信系统负责连接与数据传输，感知系统负责测量与探测，而决策与控制通常在更高层由专门系统完成。只要系统之间的接口足够清晰、任务边界足够明确，这种“功能解耦”的设计方式就能够获得较好的模块化收益，并降低系统设计复杂度。然而，随着未来无线网络应用形态的变化，这种分工逻辑正在逐步遇到根本性挑战\cite{Liu2022ISAC,Saad2020Vision6G}。

面向 6G 及未来网络的许多关键场景，不再满足于单纯的“信息传输”或单纯的“环境探测”，而是要求系统同时具备信息传输能力、环境理解能力以及实时反馈与协同决策能力，并使这三种能力在统一系统框架下形成紧密联动\cite{Gonzalez2024integrated,IMT2030Framework}。以自动驾驶为例，车辆不仅需要实时感知周围车辆、行人、障碍物和道路边界等环境要素，还需要通过 V2X 链路与其他车辆、路侧基础设施和边缘计算节点共享信息，并据此完成路径规划、风险预测和控制执行。在这一过程中，感知结果会直接影响通信内容的优先级、链路配置与资源调度，而通信传递的信息又会反过来扩展本地感知的空间范围、补偿单车视角的局限，并提升整体场景理解的完整性\cite{Liu2022ISAC,Zhang2020perceptive}。因此，感知与通信在这类场景中已经不再是两个可以先各自完成、再简单拼接的独立任务，而是一个统一业务闭环中相互依赖、相互强化的关键环节。
同样的逻辑也适用于工业互联网、低空协同、智慧交通、机器人协作和数字孪生等场景\cite{Saad2020Vision6G,Khan2022digital}. 在这些场景中，系统所面对的不是单一的“传输任务”或“探测任务”，而是围绕物理世界状态感知、信息交互、联合决策和实时控制构成的综合任务链。若仍采用“独立通信系统 + 独立感知系统 + 外部决策系统”的松耦合架构，那么系统将不可避免地面临接口复杂、时延累积、信息冗余、资源重复占用以及全局优化困难等问题。尤其是在需要毫秒级甚至更低时延闭环、并且要求环境感知和链路调度高度协同的场景中，这种松耦合模式很难满足未来网络对实时性、一致性和资源效率的要求\cite{Liu2022ISAC,Cui2024integrated}。

ISAC 的核心价值正是在这一背景下显现出来。它并不仅仅意味着“在通信系统中额外加入一个感知功能”，也不仅仅意味着“让同一套硬件顺便完成两个任务”，而是意味着打通感知、通信与业务决策之间原本相互分离的链条，使无线网络从单纯的信息传输管道升级为能够感知环境、理解状态并支撑任务闭环的综合服务平台\cite{Liu2022ISAC,Gonzalez2024integrated}. 这种价值具有明显的系统级特征：通信不再只是感知结果的搬运通道，感知也不再只是网络外部的独立输入源，二者通过统一设计形成相互赋能的能力体系，并共同作用于资源调度、链路管理、任务编排和业务执行等多个层面。
也正因如此，ISAC 不能被简单理解为对现有功能的“可选附加”。如果仅仅在通信系统旁边增设一套独立感知设备，或在感知系统之外额外接入通信链路，虽然在功能上似乎也能实现“既通信又感知”，但这种松散拼接通常难以实现统一时频空资源管理、统一协议调度和统一业务编排，更难以满足未来应用对低时延、高一致性和高资源利用率的要求\cite{Liu2020JointRadarComm,Liu2022ISAC}. 从本质上看，ISAC 的不可替代性不在于它“多做了一件事”，而在于它改变了系统组织能力的方式：它将原本彼此分离的通信、感知和部分决策支撑能力整合进一个统一框架中，从而实现传统分立系统难以达到的系统级协同收益

\subsubsection{The Core Value of System-Level Benefit Restructuring}
\label{subsubsec:ch1-isac-core-value-benefit}

理解 ISAC 的价值时，一个常见误区，是试图用某一个单一性能指标来直接判断其优劣，例如认为 ISAC 的价值在于“感知精度必须超过独立雷达”，或者“通信速率必须超过独立通信系统”。这种判断方式并不准确，甚至在很多情况下并不成立。事实上，如果只从单一功能出发进行比较，那么专门为感知任务设计的独立雷达系统，往往可以在探测距离、参数估计精度、目标检测性能或抗杂波能力等方面优于 ISAC 系统中的感知模块；同样，专门为数据传输设计的独立通信系统，也可能在吞吐量、误码率、调制编码灵活性或链路优化能力等方面优于 ISAC 系统中的通信模块\cite{Goldsmith2005,Liu2022ISAC,SkolnikRadarBook}. 这是因为独立系统能够将全部频谱、功率、时间和处理资源集中用于单一目标，而 ISAC 系统则必须在统一资源约束下同时兼顾通信与感知两类任务，其设计本质上是一个多目标折中与协同优化问题。

因此，ISAC 的真正价值并不主要体现为某一单项能力对传统专用系统的“全面超越”，而是体现为对系统整体收益结构的重新组织与重构。换言之，ISAC 改变的不是某一个局部性能指标，而是系统在资源占用方式、功能组织方式和业务支撑方式上的整体收益函数\cite{Liu2022ISAC,Gonzalez2024integrated}. 这一点可以理解为：如果传统独立系统的优化目标是分别追求“通信系统最优”和“感知系统最优”，那么 ISAC 的优化目标则是追求在统一平台上实现“综合业务效能最优”。从系统工程角度看，这种转变意味着评价标准也必须从单功能最优转向系统级最优。
更具体地说，ISAC 的系统级收益重构主要体现在以下几个方面。

\begin{enumerate}[label=(\arabic*)]
\item \textbf{Overall improvement in resource utilization.}  
传统模式下，通信系统和感知系统通常分别占用各自的频谱、硬件平台、时间资源和部署空间，导致资源割裂和重复投入。而在 ISAC 框架下，同一段频谱、同一套射频前端、同一组天线阵列、甚至同一个时频资源单元，都可以被联合用于通信与感知任务，从而显著提升系统层面的资源利用效率\cite{Liu2020JointRadarComm,Liu2022ISAC,Hassanien2015dual}. 这种提升不应简单理解为“做两件事只花一份资源”，更准确地说，它体现为通过联合设计减少资源冗余、缓解频谱紧张，并为未来高密度网络中的多任务协同提供可能。

\item \textbf{Significant reduction in total system cost.}  
ISAC 的另一项核心价值，在于通过硬件共享、站址复用、协议栈协同和网络基础设施复用，降低系统建设与运维的总体成本. 如果通信与感知分别建设独立平台，不仅需要重复配置射频链路、天线、同步模块、计算平台和供电系统，还会带来额外的部署、标定、维护和升级成本。相较之下，ISAC 倾向于依托已有蜂窝网络、车联网基础设施或局域无线平台演进式地引入感知能力，这使其在大规模部署时更具经济可行性。需要指出的是，这里的“成本降低”并不意味着单设备设计一定更简单；在许多情况下，ISAC 收发机与算法设计反而更复杂。但从系统总拥有成本（total cost of ownership）和长期规模化部署角度看，融合架构往往具有更明显优势\cite{Liu2022ISAC,Wen2024survey}。

\item \textbf{Enhanced deployment flexibility and scalability.}  
由于 ISAC 可以直接依托现有通信网络基础设施提供感知能力，因此它为新业务的快速上线和大范围扩展提供了更高的部署灵活性. 例如，在蜂窝网络、Wi-Fi 网络、车路协同网络或低空通信网络中，若节点本身 already 具备较强的信号发射、接收和多天线处理能力，则通过协议升级、波形重构和算法增强，便有可能在不完全重建硬件系统的前提下引入环境感知功能\cite{Ma2022WiFiSensingSurvey}. 这意味着，ISAC 不仅是一种“性能优化技术”，也是一种“能力演进机制”：它使无线网络能够从单一连接平台逐步演化为兼具连接与环境认知能力的综合信息基础设施。

\item \textbf{Qualitative transformation of service capability.}  
从更深层次看，ISAC 最具不可替代性的价值，体现在其推动网络服务能力发生了质变。传统通信系统主要提供信息传输能力，传统感知系统主要提供环境测量能力，而 ISAC 则使网络能够在统一框架下形成“感知—通信—决策—反馈”的闭环链条. 这一闭环能力意味着网络不再只是被动地转发数据，也不再只是孤立地生成感知结果，而是能够围绕任务目标动态整合环境信息、链路状态和业务需求，进而支撑更高层的控制、协同和智能决策。例如，在自动驾驶、工业自动化、机器人协作、无人机编队和数字孪生等场景中，应用真正需要的并不是孤立的“高精度感知”或“高吞吐通信”，而是一个能够连续感知、及时共享、快速决策并实时反馈的统一业务闭环\cite{Gonzalez2024integrated,Saad2020Vision6G}. 正是在这一意义上，ISAC 所带来的并不是某一功能模块的局部增强，而是服务范式的系统性升级。
\end{enumerate}

由此可见，评价 ISAC 的价值，不能简单沿用传统通信系统或传统雷达系统中的单一指标体系，也不能仅以“是否在某一指标上超越专用系统”作为判断依据。更合理的做法，是从系统整体角度综合考察其在资源效率、总体成本、部署演进能力以及业务闭环支撑能力等方面所带来的综合收益\cite{Liu2020JointRadarComm,Liu2022ISAC,Gonzalez2024integrated}. 换言之，ISAC 的关键意义并不在于它分别把通信和感知都做到极致，而在于它在统一平台上重新定义了“什么才是更优的系统”。以下四个小节将沿着这一思路，分别讨论频谱效率、硬件与成本、部署效率以及闭环服务能力等几个维度上的不可替代价值。

\subsection{Spectrum Efficiency Advantage}
\label{subsec:ch1-isac-core-value-spectrum}
\subsubsection{The Spectrum Occupation Issue of Traditional Independent Systems}
\label{subsubsec:ch1-isac-core-value-spectrum-occupation}

频谱资源是无线系统运行的基本物理载体，也是现代信息社会中最为稀缺的战略资源之一。无论是通信、雷达、导航，还是卫星遥感、无人系统与工业无线控制，其核心功能的实现都高度依赖对有限电磁频谱资源的占用与调度\cite{Goldsmith2005}. 然而，长期以来，通信系统与感知系统在频谱管理上大多遵循彼此分离的配置思路：不同业务按照既定规则被分配到相对独立的频段，通过静态或半静态的频率隔离来避免相互干扰。这种方式在系统规模较小、业务类型较少、跨系统协同需求不强的阶段具有明显的工程合理性，因为它简化了监管、设备设计和干扰管理问题；但随着无线业务密度和系统异构性不断上升，其所带来的频谱低效问题也日益突出\cite{Gonzalez2024integrated}。

从全球频谱占用格局来看，雷达系统在多个关键频段中长期占据重要位置，尤其是在 L 波段、S 波段、C 波段和 X 波段等传统微波频段中，航空监视雷达、气象雷达、海事雷达、军用预警雷达以及车载雷达等系统都具有广泛部署\cite{SkolnikRadarBook}. 与此同时，蜂窝移动通信、无线局域网、卫星互联网以及各类新型无线接入业务对频谱的需求持续增长。特别是在 5G-Advanced 与 6G 愿景推动下，网络需要同时支撑增强移动宽带、超低时延连接、海量机器接入以及环境智能感知等多样化任务，使得频谱紧张问题进一步加剧\cite{Liu2022ISAC,Saad2020Vision6G,IMT2030Framework}. 从需求侧看，通信系统希望获得更宽的连续带宽以支撑更高的数据速率与更低时延；而从供给侧看，适合无线传播、硬件可实现且监管允许的优质频谱资源却始终有限，这使得传统“各自独占、彼此分离”的频谱分配方式越来越难以适应未来网络的发展要求。

更值得注意的是，传统独立系统对频谱的“占有”并不等同于对频谱的“高效使用”。许多雷达系统，尤其是脉冲体制雷达，具有明显的间歇式发射特征：在时间维度上，脉冲之间存在非连续的静默区间；在空间维度上，波束往往只覆盖特定方向；在任务维度上，其工作状态还会随扫描周期、探测模式和目标活动情况而变化\cite{SkolnikRadarBook}. 因而，即使某一频段在监管层面被专门划归雷达业务，也并不意味着该频段在所有时间、所有地点和所有方向上都被充分使用。类似地，通信系统的频谱利用率同样具有显著的时空不均衡特征。例如，蜂窝基站在昼夜不同时段、城市与郊区、热点区域与普通区域之间，其流量负载可能相差数倍甚至数十倍；某些已分配频段在特定区域或特定时刻的实际利用程度并不高，但这些空闲资源通常也无法被其他系统灵活接入\cite{Haykin2005CognitiveRadio}。

由此可见，传统独立系统面临的核心问题并不只是“频谱总量不足”，更在于“频谱使用方式缺乏跨系统协调”，从而导致明显的时域、频域和空域碎片化。换言之，系统之间的隔离虽然降低了共存设计的复杂性，却也使大量潜在可用资源被锁定在各自封闭的业务边界之内，难以形成面向全局任务的高效复用\cite{Liu2020JointRadarComm,Liu2022ISAC}. 从全局角度看，这种模式造成了一种典型悖论：一方面，通信系统和感知系统都在不断抱怨频谱紧张；另一方面，大量已分配频谱又在时空上处于不连续、低负载甚至闲置状态。问题的本质不在于电磁频谱绝对不存在，而在于传统管理与系统设计范式难以充分释放其动态共享潜力。

进入高频宽带时代之后，这一矛盾进一步加剧。随着通信系统向毫米波乃至太赫兹频段扩展，越来越多的新业务开始与雷达系统在频率资源上出现更强的邻近甚至重叠关系\cite{Liu2022ISAC,Gonzalez2024integrated}. 例如，在 24~GHz、28~GHz、39~GHz 以及 60--77~GHz 等频段附近，移动通信、短距高速接入、车载雷达、成像感知和工业监测等应用都表现出强烈的频谱需求。尤其在车联网和自动驾驶场景中，通信链路与车载雷达常常同时部署于相近频段，二者不仅在频率资源上彼此接近，而且在空间覆盖范围、部署位置和服务对象上也高度耦合\cite{Sturm2011WaveformDesign,Kumari2018IEEE80211adRadar}. 在这种背景下，继续依赖传统的频率刚性隔离策略，不仅会进一步压缩可用频谱空间，也会显著限制系统之间的协同潜力。可以说，频谱占用问题已经不再只是一个监管层面的资源分配问题，而正在演化为未来无线系统架构设计中的基础性矛盾。

因此，从 ISAC 的视角重新审视频谱问题，其关键并不是简单争夺更多独立频段，而是改变“独占式占用”的传统思路，转向“按需共享、联合设计、动态复用”的新范式\cite{Liu2020JointRadarComm,Liu2022ISAC,Gonzalez2024integrated}. 也正是在这一意义上，传统独立系统的频谱占用方式构成了 ISAC 产生的重要现实背景：只有当通信与感知被放入统一的时频空资源框架中进行协同设计，频谱资源才有可能从“静态分割的稀缺资产”转化为“可动态调度的综合能力基础”。这一点，正是后续讨论 ISAC 频谱效率优势的逻辑起点。

\subsubsection{The sources of ISAC's advantages in spectrum sharing and reuse}
\label{subsubsec:ch1-adv source of isac-spectrum}

ISAC 对频谱效率的提升，并不意味着简单地让通信与感知“挤在同一个频段中同时工作”。如果缺乏统一设计，仅仅把两类业务机械地放入相同频段，反而可能导致严重的相互干扰和性能下降. ISAC 的频谱优势，本质上来源于对时、频、空、码等多维资源的联合组织与复用，即在统一系统框架下，使同一份频谱资源能够同时服务于信息传输与环境感知两类任务. 从机制上看，这种优势主要来自三个层面：同一时频资源的功能复用、面向双任务需求的联合信号设计，以及网络级的动态频谱协同调度。

\paragraph{1) Functional reuse of the same time-frequency resources.}
最直接的频谱效率增益，来自同一时频资源单元的双重功能复用。在传统独立系统中，通信系统和雷达系统通常分别占用各自独立的频带与发射时隙；而在 ISAC 系统中，同一个发射信号既可以承载通信数据，也可以作为感知探测波形使用\cite{Liu2022ISAC,Hassanien2015dual,Sturm2011WaveformDesign}. 例如，在基于 OFDM 的 ISAC 系统中，同一个 OFDM 符号可以面向通信接收端传送调制信息，而其经由目标散射后的回波又可被接收端用于距离、多普勒或角度参数估计；在多天线系统中，同一个发射波束既可以为用户提供定向通信增益，也可以照射目标区域并提升空间感知能力\cite{Liu2020JointRadarComm,Gonzalez2024integrated}. 从频谱使用角度看，这意味着原本需要分别配置给通信任务和感知任务的独立资源块，现在能够在统一资源上同步完成两类功能，因此系统总体频谱需求可以显著下降。

为了直观说明这一点，可以考虑一个简化示例。假设某通信系统需要约 100~MHz 带宽以满足目标数据速率，而某感知系统也需要约 100~MHz 有效带宽以获得相应的距离分辨率。若采用传统独立部署模式，二者通常需要分别占据各自的频谱资源，总需求约为 200~MHz；而在理想化的 ISAC 共享模式下，同一个 100~MHz 信号即可同时支持通信与感知任务，从而在总量意义上减少额外频谱占用。需要强调的是，这一示例只是为了说明“共享带来的方向性收益”，并不意味着任何实际系统都能够无损地实现“带宽需求减半”。在真实系统中，通信与感知的波形偏好、功率分配方式、峰均功率比约束、参考信号需求以及性能指标并不完全一致，因此共享过程往往伴随着一定折中\cite{Liu2020JointRadarComm,Liu2022ISAC}. 但即便如此，功能复用所带来的频谱节约和额外业务承载能力，仍然是 ISAC 最重要的基础收益之一。

\paragraph{2) Deeper spectrum gains enabled by joint waveform and signal design.}
如果说功能复用体现的是“同一资源可做两件事”，那么联合信号设计体现的则是“同一资源如何更高效地同时做好两件事”。ISAC 的更深层频谱优势，来自波形、导频、帧结构、波束赋形和功率分配等设计变量的联合优化，即在信号设计之初就同时考虑通信与感知需求，而不是将两套独立设计后的信号简单叠加. 这种联合设计能够利用信号在不同维度上的可编排性，在给定带宽内同时提供可解调的信息结构和可估计的探测特征。
以 OFDM 为例，其多载波结构天然具有较好的工程兼容性，因此被广泛视为 ISAC 的重要候选波形之一. 在通信侧，OFDM 便于灵活的子载波调制、信道均衡和多用户接入；在感知侧，宽带 OFDM 信号可通过频域结构支持距离估计，并借助跨符号相位演化支持速度感知。当系统对导频图案、子载波映射、循环前缀长度和帧结构进行适当设计时，通信系统中本来就需要存在的参考信号、同步序列或训练字段，往往可以被感知模块进一步利用，从而在不完全额外占用频谱的前提下获取目标回波信息\cite{Ma2022WiFiSensingSurvey,Kumari2018IEEE80211adRadar}. 从这个意义上说，感知功能在某些场景中并不是独立“再申请一份频谱资源”，而是“搭乘”在通信信号结构之上实现的。类似地，在 MIMO-ISAC 系统中，空间波束不仅可以服务于通信覆盖和干扰抑制，还可以通过发射波束形状控制感知照射区域与角度分辨能力，从而进一步提高频谱与空间资源的综合利用效率\cite{Liu2020JointRadarComm,Liu2022ISAC}。

\paragraph{3) Dynamic spectrum coordination at the network level.}
除单链路上的资源复用与联合信号设计之外，ISAC 的频谱优势还可以进一步延伸到网络级资源调度层面。未来蜂窝网络、车联网、工业无线网络以及低空协同网络中的通信负载和感知需求，往往都具有显著的时变性、区域性和任务相关性\cite{Saad2020Vision6G,Cui2024integrated,IMT2030Framework}. 某些时刻，网络面临高吞吐业务压力而感知需求较低；另一些时刻，则可能需要优先提升环境感知覆盖、目标跟踪精度或异常事件检测能力。在 ISAC 框架下，由于通信与感知共享底层平台和资源管理机制，系统可以根据业务上下文、场景状态和服务目标，对时隙、频段、波束、发射功率和计算资源进行联合调度\cite{Zhang2020perceptive,Cui2024integrated}. 例如，在通信负载较轻的时段，可以利用更多波束驻留时间或更密集的参考信号支持环境感知；而在感知需求较弱时，则可将更多资源优先分配给数据传输任务。与传统独立系统相比，这种动态协同调度不再受制于刚性的跨系统边界，因此能够把原本碎片化、静态化的可用频谱转化为面向任务目标动态配置的统一资源池。

综上所述，ISAC 的频谱效率优势并非来源于单一层面的“共频工作”，而是来源于多层次协同机制的共同作用：在资源层面，同一时频资源被双重利用；在信号层面，波形与结构经过联合设计以减少冗余并兼顾双任务需求；在网络层面，系统可根据时空业务变化实现跨任务动态调度. 正是这些机制共同构成了 ISAC 在频谱共享与复用上的独特优势，也说明了其频谱效率收益并不是传统独立系统通过简单拼接所能等价获得的。

\subsubsection{The applicable conditions and boundaries of the spectral efficiency advantage}
\label{subsubsec:ch1-isac-core value-spectrum-boundary}

在充分认识 ISAC 频谱效率优势的同时，也必须避免一种过度理想化的理解，即认为只要通信与感知共享频谱，就必然能够无条件获得显著增益。事实上，ISAC 的频谱效率提升并不是自动产生的，而是建立在特定场景、合理架构和精细化联合设计基础上的条件性收益. 如果系统需求之间存在较强冲突，或者收发结构、干扰抑制与资源调度机制不足，那么理论上的频谱共享优势可能无法充分转化为实际性能收益，甚至在某些场景下会出现“共享代价超过共享收益”的情况。
\paragraph{1) The inherent tension between communication-oriented and sensing-oriented signal requirements.}
首先，通信与感知对信号结构的偏好并不完全一致，这构成了 ISAC 频谱效率优势的首要边界。通信系统通常希望信号能够承载尽可能多的独立信息符号，以提升频谱效率、用户速率和多用户复用能力；而感知系统，尤其是在距离--多普勒估计、目标检测与参数跟踪等任务中，往往更偏好具有良好自相关、互相关和可控模糊函数特性的波形，以获得更稳定的估计与检测性能\cite{SkolnikRadarBook}. 当同一个信号需要同时服务于通信和感知两类任务时，系统不可避免地面临双目标折中：更多的数据承载往往意味着更弱的结构可控性，而更强的感知友好结构又可能限制通信自由度和有效吞吐. 因此，ISAC 的频谱共享并不是简单将两个任务“合并到一个频段”即可完成，而是需要在速率、误码率、检测概率、估计精度等多个指标之间寻找可接受的平衡点。
这种张力在某些极端场景中会尤为突出。例如，当通信任务强调极高的频谱效率与高阶调制、且业务负载持续饱和时，系统可用于感知参考结构和冗余设计的自由度会被显著压缩；反过来，当感知任务要求超高距离分辨率、长时间积累和稳定波束驻留时，通信系统在时隙配置、带宽占用和资源调度上的灵活性又可能受到限制. 这表明，频谱共享的收益高度依赖于通信与感知需求之间是否具有足够的兼容性。若两类任务在目标函数上差异过大，则联合频谱使用所带来的收益可能被性能折中部分甚至大部分抵消。
\paragraph{2) Self-interference and transceiver implementation constraints.}
其次，自干扰和收发机实现约束是单站 ISAC 尤其难以回避的现实问题。在许多单站一体化架构中，系统需要在同频条件下同时发射通信/感知信号并接收目标回波，这与全双工通信所面临的强自干扰问题具有相似性\cite{Sabharwal2014InBandFullDuplex}. 由于发射泄漏信号通常远强于远距离目标回波，如果缺乏足够的模拟域隔离、数字域抵消和非理想硬件补偿机制，那么回波接收链路的动态范围会受到严重压缩，进而导致感知信噪比下降、弱目标检测困难甚至距离--速度估计失效\cite{Gonzalez2024integrated,Bliss2007MIMOFullDuplex}. 从这一意义上说，ISAC 的频谱共享虽然在理论上提高了资源复用效率，但它也对射频前端隔离度、线性度、相位噪声抑制和同步精度提出了更高要求。若实现代价过高或自干扰抑制能力不足，则系统实际可获得的频谱效率收益将明显低于理想分析结果。
\paragraph{3) Sensing performance requirements impose additional constraints on resource allocation.}
再次，感知任务本身的物理分辨需求会进一步约束频谱共享收益的上限。感知性能并不是一个单一维度的指标，而是同时受到带宽、观测时间、发射功率、波束宽度和天线孔径等多种因素影响。一般而言，较高的距离分辨率需要更大的有效信号带宽，较高的速度分辨率需要更长的相干观测时间，而较高的角度分辨率则需要更大的阵列孔径或更精细的波束扫描\cite{Richards2014,SkolnikRadarBook}. 然而，这些需求往往与通信系统的资源分配策略并不完全一致。例如，通信系统为了降低时延、适应快速调度和服务更多用户，往往倾向于采用较短帧长或较灵活的时隙结构；但这可能减少感知在时间维度上的相干积累机会，从而限制速度估计精度。类似地，为了增强多用户公平性和吞吐量，通信系统可能采用较分散的频谱分配方式，而这又可能削弱感知任务对连续宽带资源的需求满足程度. 由此可见，ISAC 的频谱优势不是脱离感知分辨需求而独立存在的，其实际收益必须放在具体任务指标约束下加以评估。
\paragraph{4) Interference coupling becomes more complicated in multi-user and multi-target scenarios.}
最后，在多用户、多目标乃至多基站协同场景中，频谱共享收益还会受到更复杂的干扰耦合关系限制。在单用户、单目标或理想化链路模型下，通信与感知之间的资源共享往往容易呈现出较为直观的收益；但在实际网络中，系统往往需要同时面向多个通信终端提供差异化服务，并对多个目标或多个散射体进行检测、分辨和跟踪\cite{Liu2022ISAC,Cui2024integrated}. 此时，通信用户之间存在多用户干扰，感知目标之间存在杂波、旁瓣泄漏和参数耦合，而通信与感知任务之间又会通过波束、功率和频谱占用相互影响。随着系统规模增大，这种“多用户--多目标--多资源”的耦合关系会使简单的频谱复用策略迅速失效，并要求更复杂的联合波束赋形、功率控制、导频设计和任务调度机制\cite{Liu2020JointRadarComm,Gonzalez2024integrated}. 换言之，ISAC 在复杂场景中的频谱效率优势并非必然随系统规模线性增长，反而可能因干扰管理难度上升而受到新的性能瓶颈约束。

综上所述，ISAC 的频谱效率优势应被理解为一种具有明确适用条件和边界的系统收益，而不是一种在任意场景下都自然成立的普遍结论. 只有当通信与感知需求具有一定兼容性、收发机架构能够有效抑制自干扰、感知分辨需求与通信调度机制能够较好协调，并且多用户多目标干扰得到合理管理时，频谱共享与复用的优势才可能被充分释放。也正因如此，后续章节在讨论 ISAC 波形设计、收发机设计和资源优化时，必须从定量建模出发，分析不同系统参数和场景条件下的性能边界，而不能停留在概念层面的笼统判断。

\subsection{Hardware cost advantage}
\label{subsec:ch1-isac-core value-hardware}
\subsubsection{Independent deployment of hardware duplication issue}
\label{subsubsec:ch1-isac-core value-hardware-duplication}

当通信与感知由两套彼此独立的系统分别承担时，通常需要重复建设两条较为完整的硬件链路：包括天线阵列、射频前端、模数/数模转换器（ADC/DAC）、基带处理单元、时钟与本振模块、电源与散热单元，以及机柜、铁塔、车载支架等安装平台。对于大规模网络化部署场景，这种重复建设会显著推高系统的资本性支出（CAPEX）与运维支出（OPEX）\cite{Zhang2021ISAC6G,Liu2022ISAC,ITU2030}。
从系统组成上看，传统通信设备和传统感知设备虽然目标不同，但在硬件层面存在大量同类模块。例如，蜂窝基站和毫米波雷达都依赖高频射频前端、阵列天线、波束赋形网络以及高速数字信号处理器。若采用独立部署方案，则这些高价值部件往往被重复购置、重复安装、重复供电，且分别占用站址、频谱、回传和维护资源。
以车路协同（V2X）场景为例，若每个路侧节点同时部署一套通信设备（如 C-V2X 路侧单元）和一套专用感知设备（如毫米波雷达、摄像头或激光雷达），则至少会带来以下几方面的重复投入：
\begin{itemize}
    \item \textbf{前端硬件重复：} 通信与感知设备各自配置天线、射频链路、时钟源与处理器；
    \item \textbf{基础设施重复：} 各自需要独立的安装空间、供电接口、散热设计与防护外壳；
    \item \textbf{网络接入重复：} 各自需要数据回传链路与网络管理接口；
    \item \textbf{运维体系重复：} 各自需要独立的监测、标定、维护和故障诊断流程。
\end{itemize}
在中等规模城市中，路口、杆站、停车场、隧道口等节点数量可达数千甚至更多。此时，单节点成本差异会被网络规模显著放大，使得“是否共享硬件平台”不再只是一个技术问题，而直接关系到系统是否具备可持续部署的经济性。
除初始建设成本外，独立部署还会显著提高全生命周期成本。其原因在于：第一，双套设备通常意味着更高的总功耗；第二，设备数量增多会提高故障概率与备件管理复杂度；第三，感知设备往往还需要周期性标定与环境适配，这进一步增加了维护成本。因此，在面向 6G、智能交通、智慧园区和工业互联网等大规模场景时，降低硬件重复度已成为推动 ISAC 发展的现实驱动力之一。

\subsubsection{The basic logic of ISAC hardware sharing}
\label{subsubsec:ch1-isac-core value-hardware-sharing}

ISAC 的硬件成本优势，本质上来源于\textbf{“同一套物理资源同时服务于通信与感知”}。这种共享可以发生在前端硬件、平台基础设施、信号处理链路和网络控制等多个层面。
\paragraph{1) Front-end sharing}
在前端层面，通信与感知可以共享天线阵列、波束赋形网络、射频收发链路以及部分时钟/本振资源。尤其在毫米波和太赫兹频段，通信与感知都高度依赖窄波束增益来克服严重的路径损耗，因此两者对大规模阵列、混合波束赋形架构和宽带射频前端具有天然的共性需求\cite{Zhang2021ISAC6G, Wei2023Integrated}。  
例如，在单站式 ISAC 基站中，同一套相控阵既可以形成面向用户的通信波束，也可以形成面向环境的感知照射波束；同一功率放大器、低噪声放大器和混频器链路也可在统一调度下支持双功能运行。
\paragraph{2) Baseband and algorithmic reuse}
在基带层面，通信与感知之间存在较强的算法结构相似性。以 OFDM 为例，通信链路中的同步、FFT/IFFT、信道估计、导频处理、波束训练等模块，与感知中的时延估计、多普勒估计、角度估计和环境表征之间存在明显的数学关联\cite{Liu2022ISAC,Sturm2011WaveformDesign}。  
因此，ISAC 并不一定要求为感知功能重新配置一整套专用数字处理平台，而往往可以在现有通信基带处理器之上，通过软件升级、任务调度与部分硬件增强来实现感知能力扩展。
\paragraph{3) Platform-level integration}
在平台层面，现有蜂窝基站、Wi-Fi 接入点、路侧单元（RSU）、无人机平台以及车载终端都可以作为 ISAC 的天然承载体。以蜂窝基站为例，其已具备以下关键条件：多天线或大规模 MIMO 阵列；宽带射频与高速基带处理能力；稳定的供电、散热和安装条件；到边缘云或核心网的回传链路；现成的网络管理与时钟同步机制。
这些能力与高分辨环境感知所需的核心硬件基础高度重合。因此，相比单独建设专用感知基础设施，在现有通信平台上“叠加感知能力”通常具有更低的边际成本。
\paragraph{4) Network and data sharing}
ISAC 的成本优势不仅体现在“发射与接收设备共享”，还体现在“信息链路共享”。传统独立感知系统往往需要专门的数据回传、目标融合与控制接口；而在 ISAC 中，感知结果可以直接通过通信网络上传至边缘服务器或控制中心，通信链路本身也可承载传感数据、目标状态和协同控制信息，从而减少额外的网络基础设施投入。

\subsubsection{The limitations of hardware sharing and the engineering costs}
\label{subsubsec:ch1-isac-core value-hardware-limitations}

尽管硬件共享能够显著降低总体投入，但这种共享并非没有代价。根本原因在于：通信与感知在性能目标上并不完全一致，甚至在某些硬件指标上存在明显张力。因此，ISAC 的成本优势应理解为\textbf{系统级总成本降低}，而非“每一个单独器件都更便宜”。
\begin{itemize}
\item \textbf{动态范围（dynamic range）要求更高：}  
雷达式感知中，目标回波功率通常远低于发射信号，且不同目标、不同距离、不同散射截面积对应的回波强度差异很大。接收机需要在强泄漏、强近距回波与弱远距回波并存的情况下保持稳定工作，因此常要求更高的动态范围。相比之下，通信接收机虽然也需适应不同用户的接收功率差异，但其设计中心通常不是极弱回波检测。若 ISAC 平台需要同时满足这两类需求，则 ADC 位宽、自动增益控制和前端线性设计都可能需要增强，从而带来额外成本\cite{Wei2023Integrated}。
\item \textbf{收发隔离与自干扰抑制更困难：}  
在单站 ISAC，尤其是同时发射与接收的工作模式下，发射泄漏、自干扰和平台杂波会严重掩盖目标回波。为了可靠感知，系统往往需要较强的天线隔离、模拟域抵消和数字域干扰消除机制。这些措施会增加天线布局、射频链路和校准电路的复杂度\cite{Sturm2011WaveformDesign, Hassanien2016signaling}。
\item \textbf{时钟稳定度与相位噪声约束更严格：}  
高精度距离、多普勒和微多普勒估计通常对载波稳定性、采样时钟抖动和相位噪声更加敏感。通信系统虽然也受相位噪声影响，但很多情况下可通过同步、导频和均衡进行补偿；而感知中的精细速度估计、人体活动识别和高分辨参数估计往往更依赖稳定本振。因此，ISAC 共享时钟设计通常需要向感知侧的精度需求靠拢\cite{Sen2013YouAreWhat}。
\item \textbf{功率放大器在线性度与效率之间面临折中：}  
通信系统，特别是采用 OFDM 等高峰均功率比（PAPR）波形时，对功率放大器线性度要求较高；而传统雷达更强调高效率、大能量发射，有时偏好恒模或低 PAPR 波形。ISAC 波形和硬件必须同时兼顾可通信性与可感知性，这使得功率放大器、数字预失真和波形设计之间的协同优化更为关键\cite{Sturm2011WaveformDesign, Hassanien2016signaling}。
\item \textbf{标定与一致性维护成本不可忽视：}  
共享阵列和共享射频链路虽然减少了硬件数量，但也使系统更依赖精确校准。阵元幅相误差、通道不一致、温漂和机械偏移不仅影响通信波束成形，也会直接损害感知角度估计和成像质量。因此，大规模 ISAC 部署往往需要更加系统化的在线标定与健康监测机制，这会转化为额外的工程复杂度和软件维护成本。
\item \textbf{并非所有传感器都能被“完全替代”：}  
需要指出的是，ISAC 所强调的硬件共享主要适用于\textbf{基于无线电信号的感知}，如距离、速度、角度、占用状态和粗粒度环境感知等。对于某些对语义细节、纹理、颜色或高精度三维轮廓要求极高的任务，摄像头和激光雷达仍然具有不可替代的优势。因此，在自动驾驶、机器人或工业检测中，ISAC 更可能成为多传感器系统中的关键组成部分，而不是在所有场景下完全替代其他感知硬件\cite{Wei2023Integrated}。
\end{itemize}

综上，ISAC 的硬件共享并不是“零代价共享”，而是通过\textbf{适度增加统一平台的设计复杂度}，换取\textbf{网络级、系统级和全生命周期成本的降低}。从工程经济学角度看，只要共享带来的 CAPEX/OPEX 节省大于满足双功能需求所增加的器件、校准和算法成本，ISAC 就具有明确的硬件经济性优势。后续章节将进一步讨论硬件非理想性、收发机架构以及面向实际部署的工程实现问题。

\subsection{Deployment efficiency advantage}
\label{subsec:ch1-isac-core value-deployment}
\subsubsection{Why does the future network require high deployment efficiency}
\label{subsubsec:ch1-isac-core value-deployment-why}

未来网络的演进目标已经不再局限于“提供高速连接”，而是逐步走向通信、感知、计算与控制深度融合的综合信息基础设施形态。无论是 6G 愿景、智能交通系统、低空经济，还是工业互联网与智慧城市，其共同特征都是：\textbf{业务场景更复杂、能力需求更多样、覆盖范围更广、上线节奏更快}。因此，网络不仅要“性能更强”，还必须“部署更快、扩展更灵活、升级更容易”。

首先，未来网络所面对的应用场景呈现出快速扩张和强动态变化的趋势。以低空经济为例，无人机物流、巡检、测绘、安防和应急救援等业务，不仅需要可靠无线连接，还需要对空域中的目标、轨迹和风险进行实时感知；在车路协同场景中，道路基础设施不仅要支持车辆与路侧单元之间的信息交互，还需要具备对道路参与者、障碍物和交通态势的环境感知能力；在工业互联网场景中，移动机器人、自动化产线和数字孪生系统则要求网络同时提供高可靠连接、低时延协同以及生产现场状态感知\cite{Zhang2021ISAC6G, Feng2020JointRadarCommunication, Shafi20175G}。这些场景通常具有范围广、站点多、环境复杂和业务变化快等特点，如果网络建设周期过长，则会严重制约新业务的落地速度和规模化推广能力。

其次，传统“通信系统独立部署、感知系统独立建设”的模式难以满足未来网络对敏捷部署的要求。按照这种方式，通信侧需要建设基站、接入点、回传和网管系统，感知侧则需要单独建设雷达、摄像头、激光雷达或其他专用传感设施，并分别完成选址、供电、安装、回传接入、频谱协调和运维管理。这种“烟囱式”架构虽然便于分别优化单一功能，但在大规模网络化应用中会带来明显问题：一是站址、供电、回传和安装条件需要重复建设，导致部署成本高、周期长；二是多系统之间接口异构、协同复杂，不利于统一管理和快速扩展；三是独立规划与独立审批流程容易形成工程瓶颈，进一步拉长建网周期。

再次，未来网络需要具备更强的弹性部署和按需升级能力。许多场景并不是长期固定配置，而是具有明显的临时性、阶段性或突发性。例如，大型活动保障、灾后应急通信、临时交通管制、阶段性工业产线调整、临时低空航线开放等，都要求网络能够在较短时间内实现能力开通与业务适配。如果每引入一种新能力都需要建设一套新的专用设备体系，那么网络将难以适应这种高动态业务需求。由此可见，高部署效率不仅关系到成本，更关系到未来网络的服务敏捷性、环境适应性和商业可运营性。

从网络基础设施的演进趋势看，高部署效率还与网络的软件化、虚拟化和平台化密切相关。5G 以后，网络功能已经越来越多地通过软件定义、云化部署和开放接口实现演进，这意味着网络能力的扩展不再完全依赖大规模新增硬件，而可以更多依赖已有平台上的功能叠加与灵活编排\cite{Andrews2014, Letaief2019Roadmap}。在这一背景下，ISAC 提供了一种非常符合未来网络演进逻辑的思路：\textbf{以现有通信网络为载体，将感知能力内生化为网络基础能力之一}。如果感知功能能够在已有通信基础设施上通过软件升级、参数重构和少量硬件增强快速上线，那么网络就可以在不大幅增加部署复杂度的前提下获得新的业务能力。

因此，未来网络之所以需要高部署效率，根本原因在于：应用扩张速度快、业务变化频繁、传统独立建网模式成本高且周期长，而网络能力融合化、软件化、平台化的发展趋势又为高效部署提供了现实基础。在此背景下，ISAC 所倡导的“通信基础设施兼顾感知功能”的体系思想，正成为提升未来网络部署效率的重要技术路径\cite{Liu2022ISAC, IMT2030Framework,ITU2030}。

\subsubsection{ISAC's mechanism for enhancing deployment efficiency}
\label{subsubsec:ch1-isac-core value-deployment-mechanism}

ISAC 提升部署效率的核心机制可以概括为：\textbf{依托现有通信基础设施、网络能力和运维体系，以较低增量成本快速加载感知功能}。这一机制不仅减少了硬件重复建设，也缩短了站点规划、安装调试、网络接入和业务开通的时间链条。
\paragraph{1) Reuse of existing communication infrastructure}
当前蜂窝网络已经形成了覆盖广泛、站点密集、体系成熟的基础设施。这些基础设施所具备的能力与无线感知所需的许多基础条件高度重合。因此，若能够通过软件升级、波形重构和有限硬件扩展，使基站在提供通信服务的同时具备环境感知能力，那么感知网络的建设就不必再从“站址申请—设备采购—安装施工—接入调试”的完整流程重新开始，而可以在已有网络上进行快速叠加。
从工程视角看，这种方式至少带来四方面的部署收益：
\begin{itemize}
    \item \textbf{减少新增站址需求：} 感知能力附着于既有站点，无需大量新建独立感知点位；
    \item \textbf{减少土建与配套周期：} 既有供电、散热、机柜、铁塔和回传资源可直接复用；
    \item \textbf{减少系统集成工作量：} 感知功能可接入既有网管、编排和边缘平台；
    \item \textbf{降低审批与协调复杂度：} 减少新增设备所引发的站址审批、施工协调和运维接入流程。
\end{itemize}
\paragraph{2) Unified planning for newly deployed nodes}
对于新建网络或新扩展区域，ISAC 还能够通过“统一规划、统一建设、统一开通”的方式提升部署效率。若通信与感知由同一平台承载，则站点选址、覆盖规划、供电设计、回传接入、边缘处理和网络管理可以在同一工程流程内完成，而不必针对通信网络和感知网络分别进行规划和建设。  
这种统一规划的优势不仅体现在缩短建设周期，还体现在减少跨系统协调和后期改造难度。特别是在道路、园区、港口、机场、矿区等场景中，统一部署往往比“先建通信、后补感知”更具工程可行性。
\paragraph{3) Faster service rollout}
从业务开通角度看，ISAC 显著降低了新型感知业务上线的门槛。传统模式下，若某一区域需要新增无人机监管、交通态势监测或周界环境感知能力，通常需要重新规划专用传感节点、布设回传链路，并建立单独的数据处理与运维流程。而在 ISAC 模式下，运营者可以优先在既有通信节点上激活相关功能，通过软件更新、参数配置和少量硬件增强实现能力上线。因此，ISAC 使网络能力的新增更接近于“平台升级”而非“重新建网”。这一点对于需要快速试点、快速复制和快速扩容的新兴业务尤其重要。
\paragraph{4) Deployment efficiency from software-defined evolution}
ISAC 的另一个重要部署优势在于其与软件定义网络演进路径相契合。随着无线网络越来越依赖软件配置、云化基带、开放接口和智能编排，很多原本需要通过新增专用设备实现的功能，正在转变为通过软件重构与资源调度实现\cite{Andrews2014,Letaief2019Roadmap}。  
在这一趋势下，感知能力若能够以内生网络功能的形式提供，就意味着其上线过程可更加接近通信功能扩展的方式，即：按需启用、区域开通、策略配置和逐步升级。这使得 ISAC 在部署效率上不仅优于传统独立感知方案，也更符合未来网络云化与软硬件解耦的发展方向。

当然，上述优势通常建立在一个前提之上：通信平台本身必须具备一定的感知潜力，例如足够的带宽、阵列规模、时钟稳定度和计算资源。如果目标应用对感知精度、探测范围或目标类别识别能力提出极高要求，则仍可能需要增加专用模块甚至引入其他传感器协同。因此，ISAC 在部署效率上的优势，本质上是一种\textbf{“尽可能利用已有网络平台实现快速感知能力扩展”}的系统性优势，而不是在任何场景下都完全无需额外建设。

\subsubsection{From single-node efficiency to network-level efficiency}
\label{subsubsec:ch1-isac-core value-deployment-network}

ISAC 的部署效率优势并不只体现在单个节点的建设简化上，更重要的是其在网络级能够形成显著的规模效应和演进优势。
\paragraph{1) Efficiency gain at the single-node level}
在单站层面，ISAC 的直接收益是减少设备数量、安装环节和接口复杂度。一个 ISAC 节点在理想情况下可以同时承担原本由通信基站和部分感知设备分别完成的功能，从而减少天线系统、供电接口、机柜空间和回传接入的重复配置。  
对于铁塔承载能力有限、杆站空间紧张、城市道路设备密集的区域，这种“功能融合、平台合一”的部署方式具有特别明显的工程价值。它不仅降低了单点部署负担，也使站点美观性、能耗控制和后期维护更加可控。
\paragraph{2) Native support for multi-site collaboration}
在网络层面，现有通信系统已经具备较成熟的多站协同基础，包括时间同步、频率同步、站间接口、资源协调、切换管理和负载均衡等。这些机制原本是为支持通信连续覆盖和高效调度而建立的，但它们同样为多站协同感知提供了重要支撑\cite{Zhang2021ISAC6G,Liu2022ISAC}。  
例如，多基站联合感知通常要求不同站点之间具备较高精度的时间和频率一致性，而蜂窝网络本身已经广泛采用 GNSS 授时、同步以太网（SyncE）或 IEEE 1588 精密时间协议（PTP）等机制实现站间同步。与从零开始建设一张独立协同感知网络相比，ISAC 可以直接继承这些已有机制，从而显著降低网络级协同部署难度。
\paragraph{3) Easier scaling from local trial to wide-area deployment}
ISAC 还使网络升级路径更加平滑。运营者可以先在热点区域、重点行业园区、关键路口或低空试点区域的少量站点开通感知功能，验证业务价值与技术可行性，再逐步扩展到更大范围的站点集群，最终形成区域级乃至全网级的通感一体化覆盖。  
这种渐进式演进模式比一次性大规模独立建设感知网络更容易控制风险，原因在于：可分阶段投资，降低初期资本压力；可边部署边验证，减少技术路线不确定性；可按业务需求扩张，提升资源配置效率；可复用现有运维体系，降低组织改造难度。
\paragraph{4) From engineering efficiency to ecosystem efficiency}
进一步看，ISAC 的部署效率优势不仅是工程层面的，还具有生态层面的意义。如果通信与感知能力能够在统一平台上提供，那么设备厂商、运营商、垂直行业用户和平台开发者就可以围绕统一接口、统一站点、统一数据流和统一管理体系开展协同创新\cite{IMT2030Framework,Letaief2019Roadmap}。  
这意味着未来网络的能力部署不再是“一个功能对应一套设备”的孤立扩张模式，而更可能演变为“围绕统一网络底座持续加载新能力”的平台化扩展模式。相比传统独立系统建设，这种模式更有利于形成标准化、规模化与可复制的产业链部署路径。
\paragraph{5) A necessary caveat}
需要指出的是，ISAC 的网络级部署效率优势并不意味着所有感知任务都能直接由现有通信网络无缝承担。网络拓扑、站点高度、覆盖方向、频段特性、带宽条件以及场景遮挡情况，都会影响感知效果。在一些对垂直视场、极高精度成像或全天候鲁棒性要求很高的场景中，仍可能需要与摄像头、毫米波雷达、激光雷达或其他传感器协同部署\cite{Zhang2021ISAC6G, Wei2023Integrated}。  
因此，更准确的表述应是：\textbf{ISAC 通过继承通信网络的站点基础、协同机制与运维平台，显著提升了无线感知能力的部署效率与扩展效率，但其最终收益仍取决于具体场景需求与系统设计目标。}

综上，ISAC 在部署效率上的价值体现为一个由点到网、由硬件到平台、由工程到生态的多层次优势：单节点上减少重复建设，网络层面继承既有协同机制，演进路径上支持渐进升级，产业层面促进平台化扩展。这种优势使 ISAC 尤其适合作为未来网络中“快速引入感知能力”的关键技术方向。

\subsection{Advantage in Business Closed-Loop Capability}
\label{subsec:ch1-isac-core value-closed loop}

与传统网络主要承担“信息传递”功能不同，通信感知一体化（Integrated Sensing and Communications, ISAC）不仅能够提供数据传输能力，还能够通过对外部环境、目标状态和业务对象的实时感知，为上层应用提供更直接、更及时的状态信息，从而支撑从信息获取、状态理解、任务决策到执行反馈的完整业务链路\cite{Liu2022ISAC, Zhang2021ISAC6G, ITU2030, IMT2030Framework}。  
从这一意义上说，ISAC 推动网络从“连接工具”进一步演进为“业务闭环支撑平台”，使网络不再只是承载业务数据，而能够更深度地参与业务过程本身。
所谓\textbf{业务闭环（business closed loop）}，是指系统能够围绕某一业务目标，形成“状态感知—信息传输—分析判断—控制执行—结果反馈—持续优化”的循环过程。对于未来 6G 面向的智能交通、低空经济、工业自动化、智慧园区和公共安全等场景而言，仅有高速、低时延通信已不足以支撑高自治、高实时和高可靠业务运行，网络还需要具备对物理世界的认知能力，并能够与计算和控制过程形成紧耦合协同\cite{Letaief2019Roadmap, Saad2020Vision6G, ITU2030}。
业务闭环能力之所以重要，是因为未来许多网络业务不再只是“人发起、人决策、人执行”的传统模式，而越来越多地转向“设备自主运行、系统实时协同、网络参与控制”的新模式。因此，业务闭环能力已经成为未来网络从“提供连接”走向“支撑智能业务执行”的关键标志。

\subsubsection{The limitations of traditional communication systems}
\label{subsubsec:ch1-isac-core value-closed loop-communication}

传统通信系统的主要目标是实现用户、终端、设备和业务平台之间的信息可靠传递，其设计重点通常集中在带宽、时延、可靠性、覆盖范围、连接密度和频谱效率等指标上\cite{Goldsmith2005,Andrews2014, Shafi20175G}。从通信工程角度看，这一体系已经相当成熟，并且在移动互联网、物联网和宽带接入等场景中取得了巨大成功。然而，如果从“业务闭环”而非“信息传输”的角度审视，传统通信系统仍然存在明显局限：\textbf{它擅长传递信息，但通常并不直接产生对外部物理环境的认知能力}。

具体而言，传统通信网络虽然能够承载来自摄像头、雷达、工业传感器、车载终端和应用平台的数据流，但网络自身往往并不直接获取环境状态，也难以主动识别目标位置、运动轨迹、行为变化、空间占用或事件演化等关键信息。换言之，传统通信系统更擅长回答“信息应该被传到哪里、以多快速度传输、以多高可靠性送达”，却通常无法直接回答“环境中发生了什么、目标在哪里、状态如何变化、下一步应当如何响应”等与业务理解和业务执行密切相关的问题。
从系统架构上看，传统网络中的环境认知与状态判断大多依赖网络外部的专用感知设备和独立处理平台。例如：在车路协同中，道路环境感知通常依赖摄像头、毫米波雷达、激光雷达等独立设备；工业自动化中，设备状态检测和工位识别通常依赖专用传感器网络与控制总线；低空经济中，飞行目标探测、轨迹识别和区域预警常常依赖独立监视与探测设施。
在这些体系中，通信网络主要负责上传感知数据、分发控制命令或维持链路连接，而真正的环境理解、状态估计和决策控制往往发生在网络外部的独立系统中。
这种“感知在外、通信在中、决策在后”的分离式架构，虽然在早期业务规模有限、系统功能单一时是可行的，但在面向未来高实时、高自治和大规模协同场景时，会暴露出若干结构性问题。
\paragraph{1) Long processing chain and additional latency}
传统模式下，一个完整业务过程通常需要经历“外部感知设备采集数据—经通信网络回传至边缘或云端—上层平台分析判断—生成控制命令—再通过网络下发执行”的链条。由于感知、传输、计算、决策和执行分布在多个异构系统中，链条较长，容易引入额外传输时延、处理时延和系统交互开销\cite{Letaief2019Roadmap, Saad2020Vision6G}。对于自动驾驶避障、协作机器人控制、应急事件处置等强实时场景，这种链条过长的问题会直接影响业务闭环的反应速度。
\paragraph{2) Complex interfaces and weak cross-system coordination}
传统架构中，通信系统、感知系统和控制系统往往由不同设备、不同协议和不同管理平台构成，导致接口标准不统一、数据格式异构、同步机制复杂，进而增加系统集成和跨域协同难度。  
在多系统并行运行时，即使单个系统性能较强，整体业务闭环也可能因为接口瓶颈、同步误差和信息丢失而受到限制。
\paragraph{3) Lack of global situational awareness at the network side}
由于传统网络本身不直接参与环境感知，因此网络层往往只能看到“流量状态、连接状态和资源状态”，而难以获得“目标状态、场景状态和任务状态”的全局信息。  
这意味着网络调度通常依据通信侧指标进行优化，例如吞吐量、时延和负载均衡，而难以围绕业务任务本身进行更高层次的联合优化。例如，网络可能知道某一路侧节点流量较高，却不知道附近是否出现异常行人穿越、车辆急停或低空目标入侵，从而无法真正做到面向任务的资源分配与服务保障。
\paragraph{4) Difficult to support autonomous and closed-loop control}
未来许多场景需要网络深度参与闭环控制，而不仅是作为命令转发通道。例如，工业互联网中的运动控制、智能交通中的协同避碰、低空网络中的航迹冲突管理，都要求系统具备“边感知、边分析、边决策、边反馈”的能力\cite{Saad2020Vision6G, Liu2022ISAC}。  
如果网络自身缺乏感知能力，那么业务闭环必然依赖外部系统的结果输入，从而降低闭环反应速度、稳定性和自治程度。换言之，传统通信系统天然更适合支撑“开环式信息服务”，而不完全适合支撑“强实时自治闭环业务”。
\paragraph{5) Limited capability for continuous optimization}
业务闭环不仅意味着完成一次感知和一次控制，更意味着系统能够根据执行结果持续调整资源配置和决策策略。传统通信网络由于缺乏对物理世界状态的直接观测能力，通常难以独立完成“感知结果—业务结果—网络调度策略”之间的持续反馈优化\cite{IMT2030Framework, Letaief2019Roadmap}。  
因此，在复杂业务中，网络往往只能被动响应上层应用请求，而无法主动参与业务性能的动态优化。

综上，传统通信系统的根本局限并不在于其“通信能力不足”，而在于其系统角色主要被定义为\textbf{信息管道}，而非\textbf{环境认知与业务闭环的一部分}。随着未来网络逐步走向通感算控融合，这种定位将越来越难以满足高实时、高自治和强协同应用的需求\cite{Liu2022ISAC, ITU2030, Saad2020Vision6G}。  
也正因为如此，ISAC 的价值不仅在于提高频谱效率或降低硬件重复建设，更在于它使网络首次具备了将“连接能力”和“环境认知能力”在同一体系中结合起来的可能，从而为构建真正意义上的业务闭环网络奠定基础。

\subsubsection{ISAC forms a closed loop of ``perception $-$ transmission $-$ decision $-$ feedback''}
\label{subsubsec:ch1-isac-core value-closed loop-isac}

ISAC 的一个重要系统级价值，在于它能够推动网络形成“感知—传输—决策—执行—反馈”的业务闭环，使网络从单纯的信息承载平台演进为能够参与业务运行过程的智能基础设施\cite{Liu2022ISAC, Zhang2021ISAC6G, Saad2020Vision6G, ITU2030}。  
与传统模式中“感知在网络外、通信仅负责转发、决策与控制分散在独立平台”不同，ISAC 通过将无线感知能力内生到通信网络中，使状态获取、信息共享、任务决策与执行反馈之间的链条显著缩短，从而提升系统的实时性、自治性和持续优化能力。

从系统运行过程看，ISAC 所支撑的业务闭环通常可以概括为图景上的五个环节：\textbf{感知（perception）、传输（transmission）、决策（decision）、执行（execution）和反馈（feedback）}。其核心流程如下：

\begin{enumerate}[label=(\arabic*)]
\item \textbf{感知：获取环境状态与目标信息}

ISAC 首先利用一体化信号设计、双功能波形、共享天线阵列以及网络节点协同等，在通信过程中同步获取环境与目标状态信息，例如目标的距离、速度、角度、位置、运动趋势、占用状态以及周边环境变化等\cite{Sturm2011WaveformDesign, Hassanien2016signaling, Liu2022ISAC}。  
与传统模式依赖完全独立的专用感知设备不同，ISAC 的关键思想是：\textbf{利用通信系统已有的无线信号、频谱资源和基础设施，使网络本身具备对外部物理世界的实时认知能力}。这意味着网络不再只是“知道哪些用户在通信”，而是可以进一步“知道环境中有哪些目标、它们处于什么状态、状态如何演化”，从而为业务闭环提供第一手状态输入。

\item \textbf{传输：完成状态共享与控制信息分发}

感知得到的状态信息需要在网络内部及时共享给相关节点、边缘服务器、控制中心和业务应用平台；与此同时，决策系统生成的控制命令也需要通过通信链路可靠地下发至执行对象。  因此，在闭环业务中，通信不再只是“把用户数据送到目的地”，而是同时承担以下三种功能：
\begin{itemize}
    \item \textbf{状态同步：} 将感知结果快速传送给需要做出响应的网络节点或业务平台；
    \item \textbf{协同支撑：} 支持多节点、多终端之间共享同一环境认知结果；
    \item \textbf{命令分发：} 将调度与控制信息低时延、高可靠地下发至执行对象。
\end{itemize}
这使得通信链路本身成为闭环系统中的“信息流动骨架”，将环境状态、业务逻辑和控制动作连接为一个统一过程\cite{ITU2030, IMT2030Framework}。

\item \textbf{决策：基于感知结果进行调度、规划与控制}

在获得环境状态信息后，网络侧或应用侧可以进一步进行资源调度、风险评估、路径规划、行为判决、任务编排和控制优化等操作\cite{Letaief2019Roadmap, Saad2020Vision6G}。  
例如：在车路协同中，边缘节点可根据感知到的车辆、行人和障碍物状态生成路口协同控制策略；在低空经济中，空域管理平台可依据目标轨迹和冲突风险进行航迹调整与禁飞预警；在工业自动化中，控制系统可根据设备位置与运动状态动态调整机器人任务分配和避障策略； 在公共安全中，平台可依据异常目标检测结果生成告警、联动或跟踪方案。

此时，感知信息不再停留于“被观测”层面，而是转化为资源控制、任务调度和服务优化的直接依据。  
更进一步地，在 ISAC 网络中，决策还可以不仅发生在应用层，也可以发生在网络层。例如，网络控制面可以根据目标位置和环境状态自适应调整波束、资源块分配或感知扫描策略，从而形成\textbf{面向任务目标的网络优化机制}\cite{Liu2022ISAC, Zhang2021ISAC6G}。

\item \textbf{执行：将决策转化为物理动作或网络动作}

决策生成之后，系统需要通过执行环节将其转化为实际动作。执行对象既可能是物理实体，也可能是网络本身\cite{Saad2020Vision6G}。  
前者包括车辆转向与减速、无人机航迹调整、机械臂避障、门禁联动或告警发布等；后者则包括波束重配置、链路切换、资源重调度、感知模式切换以及边缘任务迁移等。  
将“执行”单独提出的意义在于：它表明闭环能力并不是停留在“知道了什么”和“决定做什么”，而是必须落实到“真正做了什么”。从业务系统角度看，只有当决策结果能够被稳定、及时地作用于物理世界或网络系统时，闭环才真正建立起来。

\item \textbf{反馈：通过持续感知验证效果并修正策略}

控制动作执行之后，目标状态和环境条件往往会随之变化。ISAC 可以通过持续感知再次观测执行结果，并将更新后的状态反馈给网络或应用系统，用于评估控制效果、修正偏差和优化后续决策\cite{Zhang2021ISAC6G}。  
这就形成了一个动态循环过程：
\[
\text{感知} \rightarrow \text{传输} \rightarrow \text{决策} \rightarrow \text{执行} \rightarrow \text{再感知/反馈}.
\]
与开环系统相比，这种闭环机制能够更有效地抑制环境不确定性、目标运动变化和外部扰动带来的影响，从而提升业务执行的稳定性、鲁棒性和自适应能力\cite{Saad2020Vision6G}。
\end{enumerate}

因此，ISAC 所形成的闭环能力并不是简单地将通信与感知功能机械叠加，而是通过二者深度融合，使网络能够从环境认知出发，贯通状态获取、信息共享、智能决策、任务执行与反馈修正的全过程\cite{Liu2022ISAC, Feng2020JointRadarCommunication}。  
这意味着未来网络不再只是“业务数据的传输通道”，而有望成为支撑复杂业务持续运行、动态优化和自主协同的关键基础设施。对于智能交通、低空管控、工业控制和公共安全等高实时、高自治场景而言，这种闭环能力正是 ISAC 区别于传统通信系统的核心系统价值之一。

\subsubsection{The significance of business loop capability for typical applications}
\label{subsubsec:ch1-isac-core value-closed loop-applications}

ISAC 所带来的业务闭环能力，对于未来典型应用场景具有重要意义。其价值不仅体现在“网络多了一项感知功能”，更体现在网络能够更深入地嵌入业务运行过程，支撑状态感知、信息共享、任务决策、动作执行与反馈修正的连续协同，从而提升系统的实时性、自治性和协同效率\cite{Liu2022ISAC, Zhang2021ISAC6G, Saad2020Vision6G, ITU2030}。  
对于智能交通、低空经济、工业自动化和公共安全等场景而言，很多关键业务并不是一次性的信息交换，而是一个持续演化、动态调整、反复反馈的闭环过程。因此，闭环能力往往比单纯的高速通信能力更能决定业务系统的实际运行效果。
从系统层面看，业务闭环能力的意义主要体现在以下三个方面：  
\textbf{第一，提升实时响应能力。} 由于感知、传输和控制能够在统一网络体系内联动完成，系统可以缩短从状态变化到控制响应的链条长度；  
\textbf{第二，增强自治运行能力。} 当网络能够直接获取环境状态并参与决策时，很多业务不再完全依赖人工介入或外部独立系统；  
\textbf{第三，降低跨系统协同成本。} 闭环能力使通信、感知和控制在更统一的平台上完成，有助于减少接口割裂、信息孤岛和多系统协调开销。  
这一点在以下典型应用中表现得尤为明显。
\begin{itemize}
\item \textbf{车路协同场景：支撑从信息互联走向协同驾驶闭环。}  
在智能交通与车路协同场景中，道路环境感知是驾驶决策和交通控制的重要前提。传统车联网主要面向车--车（V2V）、车--路（V2I）之间的信息交换，但车辆是否能够安全通行、是否需要减速避让、是否存在遮挡目标或盲区风险，往往仍主要依赖车载摄像头、激光雷达、毫米波雷达或独立部署的路侧感知设备。ISAC通过在通信网络中融合环境感知能力，使路侧单元、基站或车载节点不仅能够传输状态信息，还能够直接参与目标检测、轨迹估计和道路态势更新，从而将感知结果更快地反馈给车辆控制与交通调度模块 \cite{Kumari2018IEEE80211adRadar,Ma2026integrated}.  
这种能力的意义在于：网络不再只是“消息转发者”，而成为协同驾驶链条中的“状态提供者”和“决策支撑者”。借助这一闭环，系统能够更及时地获取前方目标、道路障碍、交叉口冲突风险及交通流变化，并据此支持驾驶辅助、路径优化、车速控制和路口协同管理，进而提升行车安全性与道路通行效率 \cite{Ma2026integrated,Feng2020JointRadarCommunication}。
\item \textbf{低空经济场景：支撑空域感知、调度与控制的一体化闭环。}  
在低空经济场景中，无人机配送、巡检、航测、安防和应急救援等业务对网络闭环能力提出了更高要求。此类业务通常具有目标数量多、飞行路径动态变化快、空域冲突风险高、控制时效要求强等特点，因此系统不仅需要完成飞行器与地面平台之间的通信连接，还需要持续掌握飞行器的位置、速度、航向、航迹及周边空域动态 \cite{Jiang2025integrated,Meng2023uav}.  
ISAC所提供的业务闭环能力，可以将\emph{目标探测、状态回传、轨迹预测、冲突规避、指令下发和飞行反馈}有机联结起来，使网络在低空目标识别、航迹监管、空域调度和任务协同中发挥更加主动的作用。与传统“感知系统独立运行、通信系统单独保障”的方式相比，ISAC更有利于降低系统协同成本、提升空域管理效率，并为低空业务的大规模、规范化和安全化运行提供基础支撑 \cite{Meng2023uav,Jiang2025integrated}。
\item \textbf{工业自动化场景：支撑生产监测、控制执行与反馈优化闭环。}  
工业自动化系统的本质特征之一，就是其运行过程高度依赖“监测—分析—控制—反馈”的快速闭环。现代工厂不仅要求网络连接设备、传输数据，还要求网络能够理解设备运行状态、生产节拍、作业区域变化及环境扰动，并在异常发生时及时触发控制反馈 \cite{Akyildiz2020Industrial6G}.  
ISAC能够利用统一的无线基础设施同时完成设备定位、运动状态感知、微小振动检测、区域入侵识别以及工业数据传输，从而为生产调度、异常告警、协作机器人控制和工艺优化提供更直接的状态支撑 \cite{Kaushik2024integrated,Sturm2011WaveformDesign}. 这意味着网络不再只是工业信息流的传输通道，而逐步成为连接“物理生产过程”和“数字控制系统”的关键桥梁。特别是在柔性制造、移动机器人协同、数字孪生车间等应用中，ISAC的闭环能力有助于提升系统自治水平、缩短控制链路时延，并增强复杂工业环境下的鲁棒性与可重构能力 \cite{Kaushik2024integrated,Akyildiz2020Industrial6G}。
\item \textbf{公共安全场景：支撑态势感知、风险预警与联动处置闭环。}  
公共安全业务通常具有突发性强、场景复杂、参与主体多和响应时间敏感等特点。无论是重点区域安防、人员聚集监测、周界入侵检测，还是灾害预警、搜救协同和突发事件处置，都依赖系统及时发现异常、快速共享信息并组织多方协同行动 \cite{Ma2022WiFiSensingSurvey,Liu2022ISAC}.  
在此类场景下，ISAC通过融合感知与通信能力，可更高效地支撑目标监测、行为识别、态势判断、风险预警和跨部门联动响应。网络既承担事件信息上报和处置指令下发功能，也能够直接参与环境感知和状态更新，从而形成“异常发现—信息汇聚—协同决策—行动反馈”的业务闭环。与传统分离式架构相比，这种闭环机制更有助于提升系统的实时性、覆盖弹性和整体响应效率，尤其适用于大型活动保障、重点区域监控和灾后应急通信感知一体化等任务 \cite{Ma2022WiFiSensingSurvey,Zhang2021ISAC6G}。
\item \textbf{智能家居与智慧空间场景：支撑从连接设备走向理解用户与环境的服务闭环。}  
除上述场景外，智能家居、智慧办公和智慧医疗照护等室内场景同样体现了ISAC业务闭环能力的重要性。传统无线网络主要负责设备接入和业务数据传输，而对人体存在、活动轨迹、行为模式、跌倒事件、空间占用状态等信息的获取，通常依赖额外传感器或摄像设备。近年来，基于Wi-Fi、毫米波和新一代蜂窝网络的无线感知技术表明，通信信号本身可以用于人体活动识别、呼吸与姿态监测、空间状态理解等任务 \cite{Ma2022WiFiSensingSurvey,Zhang2020device}.  
当ISAC能力引入智能空间后，网络可以将用户行为感知、状态上传、服务触发和效果反馈整合为连续闭环。例如，系统可根据室内人员位置与活动状态自动调节照明、安防、空调与健康监测策略，并通过后续感知结果不断修正服务输出。这说明ISAC的价值不仅在于提升“连接能力”，更在于使网络具备面向人、设备与环境的上下文理解能力，从而支撑更加主动、个性化和自治化的空间服务 \cite{Zhang2020device}。
\end{itemize}

总体来看，在上述典型应用中，业务闭环能力的根本意义在于：它使网络能够更深入地嵌入业务流程本身，推动系统实现从“\textbf{看见环境}”到“\textbf{理解环境}”、从“\textbf{传递信息}”到“\textbf{驱动决策}”、再从“\textbf{执行动作}”到“\textbf{持续优化}”的能力升级. 从教学和方法论角度看，这也正是ISAC区别于传统“通信系统+感知系统简单拼接”的关键所在：其目标不是功能并置，而是通过统一的信号、资源与系统设计，构建面向任务执行的闭环智能基础设施。

\subsubsection{The business closed-loop capability is the key for ISAC to achieve its 6G inherent capabilities}
\label{subsubsec:ch1-isac-core value-closed loop-6g}

从技术演进与网络体系升级的角度看，\textbf{业务闭环能力}是ISAC从“功能集成”走向“6G内生能力”的关键环节。对于ISAC而言，只有当感知能力能够持续参与业务流程，并在网络运行、资源组织、任务执行和反馈优化中发挥稳定作用时，通信与感知的融合才具有真正的体系化意义 \cite{Liu2022ISAC,Zhang2021ISAC6G,Saad2020Vision6G}。

首先，\textbf{业务闭环能力使感知从附属测量功能演进为网络服务能力的重要组成部分}。在传统通信系统中，网络的核心任务主要是信息传输，而测距、定位、目标探测和环境观测等能力通常被视为附加功能，更多服务于特定垂直应用或外部感知系统 \cite{Sturm2011WaveformDesign}. 然而，在ISAC框架下，感知结果不再只是“被动提供的环境信息”，而是直接影响链路建立、波束管理、无线资源调度、移动性管理、任务编排以及应用控制的重要依据 \cite{Kaushik2024toward}.  
这意味着，感知的角色已从传统意义上的“补充测量”转变为面向网络自治和业务优化的基础能力。换言之，感知不再是“网络可选的附加功能”，而是网络提供高可靠、低时延、环境自适应和任务驱动服务时必须具备的组成部分。对于6G而言，这种转变具有标志性意义：网络不仅需要传输比特（bits），还需要理解场景（contexts）、感知状态（states）并支撑行动（actions）\cite{Saad2020Vision6G,Strinati20216G}。

其次，\textbf{业务闭环能力是ISAC走向网络内生智能与系统自治的重要基础}。6G网络普遍被认为将从“连接型网络”向“智能型网络”演进，其目标不仅在于提升峰值速率和连接密度，更在于增强网络对物理环境、用户行为和业务任务的理解能力，从而实现更高程度的自适应、自优化和自管理 \cite{Saad2020Vision6G,Strinati20216G}. 在这一演进过程中，ISAC的核心价值并不只体现在频谱共享或硬件复用，更在于它使网络能够直接感知外部物理世界，并将这种感知结果纳入网络控制闭环之中。  
例如，网络可基于目标位置和运动状态动态调整波束指向与接入策略，基于环境变化优化链路配置与切换决策，或基于空间态势感知结果触发更有针对性的资源分配和服务保障机制 \cite{Liu2022ISAC,Zhang2021ISAC6G}. 如果缺乏这种闭环机制，感知信息往往只能停留在“辅助观测”的层面，难以真正转化为网络自治所需的决策变量和控制输入。因此，业务闭环能力实际上决定了ISAC能否从“可集成的技术模块”上升为“可内生于网络体系的基础能力”。

再次，\textbf{业务闭环能力为“ISAC+”以及“通感算控一体化”提供了逻辑起点与体系接口}。未来6G的重要发展方向之一，是推动通信（Communication）、感知（Sensing）、计算（Computation）与控制（Control）的深度协同，以支撑智能制造、自动驾驶、低空协同、数字孪生和人机交互等复杂任务型业务. 在这一过程中，如果ISAC仅停留在信号层或物理层的一体化设计，其作用主要体现为波形复用、频谱共用和硬件共享，尽管重要，但仍属于局部优化 \cite{Sturm2011WaveformDesign}.  
而一旦ISAC具备了完整的业务闭环能力，其感知结果便可自然向更高层功能模块输出：感知信息可触发边缘计算任务卸载与模型更新，计算结果可进一步支持任务理解与状态预测，控制模块则依据这些结果生成执行策略，而控制执行后的效果又可以通过感知机制进行验证和修正 \cite{Shi2016EdgeComputing,Chen2024integrated}. 从这个意义上说，业务闭环能力不仅提升了ISAC自身的应用深度，也为后续演进到“通感算控一体化”提供了结构化接口和方法论基础。

最后，\textbf{业务闭环能力决定了ISAC能否真正成为6G网络中的基础能力与核心服务能力}。6G所强调的“原生智能”“原生感知”或“内生能力”，并不只是指网络中增加若干新的功能模块，而是指这些能力能够嵌入网络运行机制本身，成为支撑业务执行与系统优化的底层要素 \cite{Saad2020Vision6G,Strinati20216G}. 对ISAC而言，若感知仅作为孤立功能存在，即使其在测距精度、目标检测或定位性能上表现优异，也仍然难以支撑网络体系层面的深度演进；只有当ISAC能够贯通感知、传输、决策与反馈的连续循环，服务于业务全流程运行，它才具备从技术概念走向网络原生能力的条件 \cite{Liu2022ISAC,Kaushik2024toward}.  

因此，可以认为：\textbf{业务闭环能力既体现了ISAC相对于传统通信系统和传统分离式感知系统的本质优势，也决定了其能否真正融入6G网络架构、成为面向未来任务型业务的基础支撑能力}。这一本质判断对于理解ISAC的研究重点具有重要启发意义：未来ISAC设计不能仅关注通信性能与感知性能的并列提升，还需要进一步关注感知信息如何进入网络决策链条、如何参与业务控制过程，以及如何在闭环运行中持续创造系统级价值。

\subsection{Reflective Questions}
\label{subsec:ch1.2-reflect-questions}

本节思考题围绕 ISAC 的系统级价值展开，重点考察读者对“系统能力重构”“系统级收益重构”以及频谱效率、硬件成本、部署效率和业务闭环能力等核心价值的理解。建议读者在完成本节学习后独立作答，其中部分题目可作为课堂讨论或课程作业选题。

\paragraph{一、概念理解类}
\begin{enumerate}
    \item 为什么说 ISAC 不是“可有可无的功能叠加”，而是一种系统能力的重构？请结合“独立通信系统”“独立感知系统”和“融合系统”的差异进行说明。

    \item 本节指出，ISAC 的核心价值不在于某一项单点性能指标的提升，而在于系统级收益的重构。请解释“系统级收益重构”的含义，并说明它与“某一项指标更优”之间的本质区别。

    \item 为什么说传统通信系统的设计目标主要面向信息传输，而非环境状态获取，因此在业务层面上常表现为“对物理世界无感”？为什么这种局限在 5G 以前可能并不突出，而在 6G 时代会变得越来越明显？请结合具体应用场景加以说明。
\end{enumerate}

\paragraph{二、对比分析类}
\begin{enumerate}
    \setcounter{enumi}{3}
    \item 请从以下四个方面，对比两种部署方案的差异，并分析哪一种更适合面向 6G 的大规模新型场景：
    \begin{enumerate}
        \item 频谱占用方式；
        \item 硬件建设成本；
        \item 部署流程与实施复杂度；
        \item 业务支撑能力。
    \end{enumerate}
    两种方案分别为：
    \begin{enumerate}
        \item 通信系统与感知系统独立部署；
        \item 采用 ISAC 的一体化部署。
    \end{enumerate}

    \item 本节从频谱效率、硬件成本、部署效率和业务闭环能力四个方面分析了 ISAC 的核心价值。请说明这四类价值是否彼此独立；如果不是，请分析它们之间可能存在的耦合关系与相互促进机制。

    \item 频谱效率提升并不意味着“任何场景下 ISAC 都一定优于独立系统”。请结合以下因素，分析 ISAC 频谱增益成立的适用条件及其边界：
    \begin{enumerate}
        \item 双功能之间的资源冲突；
        \item 互扰与自干扰；
        \item 目标分辨率要求；
        \item 通信链路质量要求。
    \end{enumerate}
\end{enumerate}

\paragraph{三、深度思考与论述类}
\begin{enumerate}
    \setcounter{enumi}{6}
    \item 假设某城市计划建设车路协同基础设施，有两种备选方案：

    方案 A：分别建设通信基站和路侧雷达；

    方案 B：在通信基站上升级 ISAC 能力。

    请从 CAPEX、OPEX、部署周期、维护复杂度和后续业务扩展能力五个角度，对两种方案进行比较，并给出你的建议。

    \item 本节提到，ISAC 通过共享天线阵列、射频链路和基带处理平台等硬件资源，能够降低系统总成本。请思考：尽管通信与感知在动态范围、线性度、时钟精度和收发隔离等方面的需求并不完全一致，为什么硬件共享在很多场景下仍可能是总体上更经济的方案？请从“系统总成本”而非“单模块最优性能”的角度进行论述。

    \item 请围绕“部署效率”这一价值，回答以下问题：
    \begin{enumerate}
        \item 为什么说 ISAC 特别适合需要快速、广域和规模化部署的低空经济、工业互联网和公共安全等新兴场景？
        \item 如果这些场景完全依赖独立感知基础设施，在部署上可能会遇到哪些现实障碍？
    \end{enumerate}

    \item 本节提出，ISAC 的最大价值之一在于形成“感知—传输—决策—反馈”的业务闭环。请选择以下任一场景进行分析：车路协同、低空无人机管理、智能工厂或公共安全监测。要求说明：
    \begin{enumerate}
        \item 感知环节获取什么信息；
        \item 通信环节传输什么内容；
        \item 由谁完成决策；
        \item 反馈如何作用于下一轮系统行为。
    \end{enumerate}

    \item 有一种观点认为：“只要在通信系统外接一些雷达或传感器，也能实现环境感知，因此没有必要发展 ISAC。”请结合本节内容，对这一观点进行评述。要求至少从以下三个方面展开：
    \begin{enumerate}
        \item 系统成本；
        \item 部署效率；
        \item 业务闭环能力。
    \end{enumerate}

    \item 请论述为什么说业务闭环能力是 ISAC 走向 6G 内生能力的关键，而不仅仅是一个附加优势。提示：可从网络角色变化、智能化需求、数字孪生以及“通感算控”一体化等角度展开。

    \item 假设未来 6G 网络已普遍具备 ISAC 能力，那么运营商的角色将发生怎样的变化？其是否仍主要扮演“连接服务提供者”的角色，还是会进一步演进为“环境信息服务提供者”乃至“决策支撑平台”？请谈谈你的理解。
\end{enumerate}

\paragraph{四、开放探索类}

    \setcounter{enumi}{13}
    \item 如果你是一名系统设计者，需要为某一类应用场景评估 ISAC 的实际价值，你会从哪些维度建立评估指标体系？请至少给出 5 个评价维度，并说明每个维度分别对应 ISAC 的哪一类核心价值。构建指标体系时，应尽量兼顾技术性能、部署成本与业务价值。
