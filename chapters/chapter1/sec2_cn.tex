上一节主要从概念与演进逻辑的角度回答了“ISAC 是什么”以及“ISAC 如何逐步发展而来”的问题。本节进一步讨论一个更为根本的问题：\emph{ISAC 的核心价值究竟是什么，以及为什么这种价值并不是传统独立系统简单叠加所能够替代的？} 我们将从频谱效率、硬件成本、部署效率和业务闭环能力等几个维度展开分析，揭示 ISAC 相较于传统独立通信系统和独立感知系统所具有的系统级优势。更重要的是，本节将说明，这些优势并不是彼此孤立的局部收益，而是共同指向一种网络能力形态的根本变化：无线网络不再只是承担“传输信息”的单一角色，而开始具备“感知环境、理解状态、支撑决策”的综合服务能力\cite{Liu2022ISAC,Gonzalez2024integrated,Saad2020Vision6G}。正是在这一意义上，ISAC 不应被理解为一种可有可无的功能附加，而应被理解为未来 6G 网络的重要内生能力之一。

\subsection{Why is ISAC not merely an "optional addition to existing functions"?}
\label{subsec:ch1-isac-core-value-why}
\subsubsection{The value logic from independent systems to integrated systems}
\label{subsubsec:ch1-isac-core-value-logic}

传统上，通信系统与感知系统沿着相互独立的路径发展。通信系统以可靠、高效地传输信息为核心，重点关注频谱效率、覆盖、时延和可靠性等指标\cite{Goldsmith2005}；感知系统则以理解物理环境为目标，关注目标检测、参数估计、状态跟踪和场景感知等能力\cite{SkolnikRadarBook}。二者分别承担“传信息”和“看环境”的任务，这种功能解耦在过去较长时期内具有良好的模块化优势。

然而，面向 6G 的许多新型场景，如自动驾驶、工业互联网、低空协同和数字孪生，不再只需要单一的通信或感知能力，而是要求系统同时具备环境感知、信息交互和实时决策支撑能力，并在统一框架下协同工作\cite{Saad2020Vision6G,Gonzalez2024integrated,IMT2030Framework}。以自动驾驶为例，车辆既要感知周围目标和道路状态，又要通过 V2X 与其他车辆和基础设施共享信息；感知结果会影响通信调度，而通信又能扩展感知范围、提升场景理解能力\cite{Zhang2020perceptive}。此时，通信与感知已不再是两个可简单拼接的独立模块，而是统一业务闭环中相互依赖的关键环节。

如果仍采用“独立通信系统 + 独立感知系统 + 外部决策系统”的松耦合架构，往往会带来接口复杂、时延累积、资源重复占用和全局优化困难等问题，难以满足未来网络对低时延、高一致性和高资源效率的要求\cite{Cui2024integrated,Liu2020JointRadarComm}。

因此，ISAC 的价值并不在于“额外增加一个感知功能”，也不只是“让同一套硬件执行两项任务”，而在于通过统一设计打通通信、感知与业务决策之间的链条，使无线网络从单纯的信息传输平台演进为兼具环境理解与任务支撑能力的综合服务平台\cite{Liu2022ISAC,Gonzalez2024integrated}。从本质上看，ISAC 的不可替代性不在于它“多做了一件事”，而在于它改变了系统组织能力的方式，从而实现传统分立系统难以达到的系统级协同收益。

\subsubsection{The Core Value of System-Level Benefit Restructuring}
\label{subsubsec:ch1-isac-core-value-benefit}

理解 ISAC 价值时，一个常见误区是用单一指标直接判断其优劣，例如要求其感知精度超过独立雷达，或通信速率超过独立通信系统。事实上，这种比较往往并不合理。专用雷达通常在探测距离、估计精度和检测性能等方面更具优势，专用通信系统也可能在吞吐量、误码率和链路优化能力等方面表现更好\cite{Goldsmith2005,Liu2022ISAC,SkolnikRadarBook}。原因在于，独立系统可将全部资源集中于单一任务，而 ISAC 需要在统一资源约束下同时兼顾通信与感知，本质上是一个多目标折中与协同优化问题。

因此，ISAC 的核心价值并不在于单项性能全面超越专用系统，而在于重构系统整体收益。与传统系统分别追求“通信最优”或“感知最优”不同，ISAC 更强调在统一平台上实现“综合业务效能最优”\cite{Liu2022ISAC,Gonzalez2024integrated}。其系统级收益主要体现在以下几个方面：

\begin{enumerate}[label=(\arabic*)]
\item \textbf{提升资源利用效率。}
通信与感知可共享频谱、硬件平台、天线和时频资源，从而减少资源冗余，缓解频谱紧张，提高系统整体资源利用率\cite{Liu2020JointRadarComm,Liu2022ISAC,Hassanien2015dual}。

\item \textbf{降低系统总体成本。}
通过硬件共享、站址复用和基础设施协同，ISAC 有助于降低建设与运维成本。虽然融合系统的设计可能更复杂，但从总拥有成本和规模部署角度看通常更具优势\cite{Liu2022ISAC,Wen2024survey}。

\item \textbf{增强部署灵活性与可扩展性。}
ISAC 可依托现有蜂窝、Wi-Fi、车联网等通信基础设施，通过协议升级、波形重构和算法增强引入感知能力，从而支持新业务的快速部署与网络能力演进\cite{Ma2022WiFiSensingSurvey}。

\item \textbf{推动服务能力从“连接”走向“闭环智能”。}
ISAC 使网络能够在统一框架下形成“感知—通信—决策—反馈”闭环，不仅传输信息，也支持环境认知、协同控制和智能决策。这种能力对自动驾驶、工业自动化、机器人协作和数字孪生等场景尤为关键\cite{Gonzalez2024integrated,Saad2020Vision6G}。
\end{enumerate}

因此，评价 ISAC 不能仅看某一单项指标是否超过专用系统，而应从系统整体角度考察其在资源效率、成本、部署演进和业务闭环支撑等方面的综合收益。ISAC 的关键意义，不在于分别把通信和感知做到极致，而在于在统一平台上重新定义更优的系统形态。

\subsection{Spectrum Efficiency Advantage}
\label{subsec:ch1-isac-core-value-spectrum}
\subsubsection{The Spectrum Occupation Issue of Traditional Independent Systems}
\label{subsubsec:ch1-isac-core-value-spectrum-occupation}

频谱资源是无线系统运行的基础载体，也是极为稀缺的战略资源。通信、雷达、导航、遥感以及工业无线控制等系统都依赖有限频谱来实现核心功能\cite{Goldsmith2005}。长期以来，通信与感知系统通常采用彼此独立的频谱配置方式，即通过静态或半静态的频段划分来避免相互干扰。这种方式在早期具有实现简单、监管清晰的优点，但随着无线业务密度提升和系统异构化加剧，其频谱利用低效的问题日益突出\cite{Gonzalez2024integrated}。

从实际占用情况看，雷达系统长期部署于 L、S、C、X 等关键频段，而蜂窝通信、无线局域网、卫星互联网及新型无线接入业务的频谱需求也在持续增长\cite{SkolnikRadarBook}. 在 5G-Advanced 和 6G 背景下，网络不仅要支持高速率、低时延和大连接，还要具备环境感知能力，这进一步加剧了频谱紧张\cite{Liu2022ISAC,Saad2020Vision6G,IMT2030Framework}。因此，传统“各自独占、彼此分离”的分配模式已难以适应未来无线网络的发展需求。

更重要的是，频谱“被分配”并不意味着“被高效利用”。雷达系统常具有间歇发射、定向波束和任务驱动的工作特点，其频谱占用在时间、空间和任务维度上往往是不连续的\cite{SkolnikRadarBook}。通信系统同样存在明显的时空负载不均衡现象，例如不同区域、不同时段的业务需求差异显著，导致部分已分配频谱在特定场景下利用率较低\cite{Haykin2005CognitiveRadio}。因此，问题的核心不仅是频谱总量有限，更在于传统独立系统缺乏跨系统协调，造成频谱资源在时域、频域和空域上的碎片化\cite{Liu2020JointRadarComm,Liu2022ISAC}。

这一矛盾在毫米波和太赫兹等高频宽带场景下更加突出。随着通信与感知业务向相近频段扩展，双方在 24~GHz、28~GHz、39~GHz 以及 60--77~GHz 等频段的频谱需求不断重叠\cite{Liu2022ISAC,Gonzalez2024integrated}。特别是在车联网与自动驾驶场景中，通信链路与车载雷达不仅频段接近，而且部署位置、覆盖区域和服务对象也高度耦合\cite{Sturm2011WaveformDesign,Kumari2018IEEE80211adRadar}。在此背景下，继续依赖刚性的频率隔离策略，将限制频谱利用效率和系统协同能力。

因此，ISAC 对频谱问题的重新审视，不在于争取更多独立频段，而在于从“独占式占用”转向“按需共享、联合设计与动态复用”的新范式\cite{Liu2020JointRadarComm,Liu2022ISAC,Gonzalez2024integrated}。只有将通信与感知纳入统一的时频空资源框架进行协同设计，频谱资源才可能从静态分割的稀缺资产转化为可动态调度的综合能力基础。这正是 ISAC 频谱效率优势的现实起点。

\subsubsection{The sources of ISAC's advantages in spectrum sharing and reuse}
\label{subsubsec:ch1-adv source of isac-spectrum}

ISAC 的频谱效率提升，并不是简单地让通信与感知在同一频段“同时工作”。如果缺乏统一设计，机械共频反而可能带来严重干扰。其核心优势在于：在统一框架下，对时、频、空、码等资源进行联合组织与复用，使同一份频谱同时服务于信息传输与环境感知。总体上，这种优势主要来自三个方面：同一资源的功能复用、面向双任务的联合信号设计，以及网络级动态频谱协同。

\paragraph{1) Functional reuse of the same time-frequency resources.}
最直接的增益来自同一时频资源的双重使用。传统系统中，通信和雷达通常分别占用独立频带与时隙；而在 ISAC 中，同一发射信号既可承载通信信息，也可作为感知探测波形\cite{Liu2022ISAC,Hassanien2015dual,Sturm2011WaveformDesign}。例如，在 OFDM-ISAC 中，同一个 OFDM 符号既可用于数据传输，也可通过目标回波实现距离、多普勒或角度估计；在多天线系统中，同一波束既可提升通信增益，也可照射目标区域以增强感知能力\cite{Liu2020JointRadarComm,Gonzalez2024integrated}。因此，原本分别分配给通信和感知的资源块，可以在统一资源上同步完成两类任务，从而降低总体频谱占用。

以一个简化例子说明：若通信和感知各自都需要约 100~MHz 带宽，传统独立部署可能总共需要约 200~MHz；而在理想 ISAC 共享模式下，同一个 100~MHz 信号可同时支持两类任务。需要指出，这只是说明共享的方向性收益，实际系统通常无法无损实现“需求减半”，因为通信与感知在波形、功率、导频和性能指标上往往存在差异\cite{Liu2020JointRadarComm,Liu2022ISAC}。尽管如此，功能复用仍是 ISAC 频谱收益的基础来源。

\paragraph{2) Deeper spectrum gains enabled by joint waveform and signal design.}
如果说功能复用回答的是“同一资源能做两件事”，那么联合信号设计回答的就是“如何更高效地同时做好两件事”。ISAC 的更深层优势来自对波形、导频、帧结构、波束赋形和功率分配的联合优化，即从设计源头同时考虑通信与感知需求，而不是简单叠加两套独立信号。

以 OFDM 为例，其多载波结构兼具通信和感知友好性。在通信侧，OFDM 便于调制、均衡和多用户接入；在感知侧，其宽带结构有利于距离估计，跨符号相位演化有助于速度感知。通过合理设计导频图案、子载波映射、循环前缀和帧结构，通信中原本用于同步和信道估计的参考信号，往往也可被感知模块复用，从而在不额外占用太多频谱的情况下获得目标信息\cite{Ma2022WiFiSensingSurvey,Kumari2018IEEE80211adRadar}。类似地，在 MIMO-ISAC 中，空间波束既服务于通信覆盖和干扰控制，也影响感知照射范围与角度分辨能力，从而提高频谱与空间资源的综合利用效率\cite{Liu2020JointRadarComm,Liu2022ISAC}。

\paragraph{3) Dynamic spectrum coordination at the network level.}
除了单链路上的资源复用与联合设计，ISAC 的频谱优势还体现在网络级调度上。未来蜂窝、车联网、工业无线和低空网络中的通信负载与感知需求通常具有明显的时变性和区域性\cite{Saad2020Vision6G,Cui2024integrated,IMT2030Framework}。在 ISAC 框架下，由于通信与感知共享底层平台，系统可根据场景状态和任务目标，对时隙、频段、波束、功率和计算资源进行联合调度\cite{Zhang2020perceptive,Cui2024integrated}。例如，通信负载较低时可增加感知驻留时间或参考信号密度；感知需求较弱时则可优先保障数据传输。相比传统独立系统，这种动态协同能够将碎片化、静态化的频谱资源转化为面向任务目标的统一资源池。

综上，ISAC 的频谱优势并非仅仅来自“共频工作”，而是来自多层次协同：资源层面的双重利用、信号层面的联合设计，以及网络层面的动态调度。也正因如此，ISAC 的频谱效率提升并不是传统独立系统简单拼接所能等价实现的。

\subsubsection{The applicable conditions and boundaries of the spectral efficiency advantage}
\label{subsubsec:ch1-isac-core value-spectrum-boundary}

尽管 ISAC 具有提升频谱效率的潜力，但这种优势并非只要共享频谱就能自动获得，而是依赖于具体场景、系统架构与联合优化设计。当通信与感知需求冲突明显，或收发机实现、干扰抑制和资源调度能力不足时，频谱共享的理论收益可能难以转化为实际性能增益，甚至出现共享代价高于共享收益的情况。

\paragraph{1) The inherent tension between communication-oriented and sensing-oriented signal requirements.}
通信与感知对信号结构的要求并不完全一致。通信更关注高频谱效率、高吞吐量和多用户复用，而感知更强调波形的自相关、互相关及模糊函数特性，以支持检测、估计与跟踪\cite{SkolnikRadarBook}。因此，同一信号同时服务两类任务时，必须在速率、误码率、检测概率和估计精度之间进行权衡。在高负载通信或高精度感知等极端场景下，这种矛盾会更加突出，从而削弱频谱共享收益。

\paragraph{2) Self-interference and transceiver implementation constraints.}
在单站 ISAC 中，同频发射与接收会带来强自干扰问题，类似于全双工通信\cite{Sabharwal2014InBandFullDuplex}。由于发射泄漏通常远强于目标回波，若模拟隔离、数字抵消和硬件补偿不足，感知链路的动态范围将受到压缩，导致弱目标检测和参数估计性能显著下降\cite{Gonzalez2024integrated,Bliss2007MIMOFullDuplex}。因此，ISAC 的频谱复用优势往往以更高的硬件实现复杂度为代价。

\paragraph{3) Sensing performance requirements impose additional constraints on resource allocation.}
感知性能受带宽、观测时间、功率和阵列孔径等因素共同制约。高距离分辨率需要更大带宽，高速度分辨率需要更长观测时间，高角度分辨率需要更大孔径或更精细波束控制\cite{Richards2014,SkolnikRadarBook}。而这些需求未必与通信系统追求低时延、灵活调度和多用户服务的资源配置策略一致。因此，ISAC 的频谱效率优势必须结合具体感知指标来评估，而不能脱离任务需求单独讨论。

\paragraph{4) Interference coupling becomes more complicated in multi-user and multi-target scenarios.}
在多用户、多目标和多基站协同场景中，通信干扰、感知杂波与旁瓣泄漏相互交织，使频谱共享问题更加复杂\cite{Liu2022ISAC,Cui2024integrated}。随着系统规模增大，简单的复用策略往往难以维持收益，需要依赖更复杂的联合波束赋形、功率控制、导频设计与任务调度机制\cite{Liu2020JointRadarComm,Gonzalez2024integrated}。因此，ISAC 的频谱效率优势并不会随系统规模线性增长，反而可能受到更强的干扰管理瓶颈限制。

综上，ISAC 的频谱效率优势是一种有条件的系统收益，而非普适结论。只有当通信与感知需求较为兼容、收发机能够有效抑制自干扰、感知分辨需求与通信调度可协调、且多用户多目标干扰得到合理控制时，这种优势才能真正体现。因此，后续对波形设计、收发机设计和资源优化的讨论，应基于定量模型分析其适用边界与性能上限。

\subsection{Hardware cost advantage}
\label{subsec:ch1-isac-core value-hardware}
\subsubsection{Independent deployment of hardware duplication issue}
\label{subsubsec:ch1-isac-core value-hardware-duplication}

当通信与感知由两套彼此独立的系统分别承担时，通常需要重复建设两条较为完整的硬件链路：包括天线阵列、射频前端、模数/数模转换器（ADC/DAC）、基带处理单元、时钟与本振模块、电源与散热单元，以及机柜、铁塔、车载支架等安装平台。对于大规模网络化部署场景，这种重复建设会显著推高系统的资本性支出（CAPEX）与运维支出（OPEX）\cite{Zhang2021ISAC6G,Liu2022ISAC,ITU2030}。
从系统组成上看，传统通信设备和传统感知设备虽然目标不同，但在硬件层面存在大量同类模块。例如，蜂窝基站和毫米波雷达都依赖高频射频前端、阵列天线、波束赋形网络以及高速数字信号处理器。若采用独立部署方案，则这些高价值部件往往被重复购置、重复安装、重复供电，且分别占用站址、频谱、回传和维护资源。
以车路协同（V2X）场景为例，若每个路侧节点同时部署一套通信设备（如 C-V2X 路侧单元）和一套专用感知设备（如毫米波雷达、摄像头或激光雷达），则至少会带来以下几方面的重复投入：
\begin{itemize}
    \item \textbf{前端硬件重复：} 通信与感知设备各自配置天线、射频链路、时钟源与处理器；
    \item \textbf{基础设施重复：} 各自需要独立的安装空间、供电接口、散热设计与防护外壳；
    \item \textbf{网络接入重复：} 各自需要数据回传链路与网络管理接口；
    \item \textbf{运维体系重复：} 各自需要独立的监测、标定、维护和故障诊断流程。
\end{itemize}
在中等规模城市中，路口、杆站、停车场、隧道口等节点数量可达数千甚至更多。此时，单节点成本差异会被网络规模显著放大，使得“是否共享硬件平台”不再只是一个技术问题，而直接关系到系统是否具备可持续部署的经济性。
除初始建设成本外，独立部署还会显著提高全生命周期成本。其原因在于：第一，双套设备通常意味着更高的总功耗；第二，设备数量增多会提高故障概率与备件管理复杂度；第三，感知设备往往还需要周期性标定与环境适配，这进一步增加了维护成本。因此，在面向 6G、智能交通、智慧园区和工业互联网等大规模场景时，降低硬件重复度已成为推动 ISAC 发展的现实驱动力之一。

\subsubsection{The basic logic of ISAC hardware sharing}
\label{subsubsec:ch1-isac-core value-hardware-sharing}

ISAC 的硬件成本优势，本质上来源于\textbf{“同一套物理资源同时服务于通信与感知”}。这种共享可以发生在前端硬件、平台基础设施、信号处理链路和网络控制等多个层面。
\paragraph{1) Front-end sharing}
在前端层面，通信与感知可以共享天线阵列、波束赋形网络、射频收发链路以及部分时钟/本振资源。尤其在毫米波和太赫兹频段，通信与感知都高度依赖窄波束增益来克服严重的路径损耗，因此两者对大规模阵列、混合波束赋形架构和宽带射频前端具有天然的共性需求\cite{Zhang2021ISAC6G, Wei2023Integrated}。  
例如，在单站式 ISAC 基站中，同一套相控阵既可以形成面向用户的通信波束，也可以形成面向环境的感知照射波束；同一功率放大器、低噪声放大器和混频器链路也可在统一调度下支持双功能运行。
\paragraph{2) Baseband and algorithmic reuse}
在基带层面，通信与感知之间存在较强的算法结构相似性。以 OFDM 为例，通信链路中的同步、FFT/IFFT、信道估计、导频处理、波束训练等模块，与感知中的时延估计、多普勒估计、角度估计和环境表征之间存在明显的数学关联\cite{Liu2022ISAC,Sturm2011WaveformDesign}。  
因此，ISAC 并不一定要求为感知功能重新配置一整套专用数字处理平台，而往往可以在现有通信基带处理器之上，通过软件升级、任务调度与部分硬件增强来实现感知能力扩展。
\paragraph{3) Platform-level integration}
在平台层面，现有蜂窝基站、Wi-Fi 接入点、路侧单元（RSU）、无人机平台以及车载终端都可以作为 ISAC 的天然承载体。以蜂窝基站为例，其已具备以下关键条件：多天线或大规模 MIMO 阵列；宽带射频与高速基带处理能力；稳定的供电、散热和安装条件；到边缘云或核心网的回传链路；现成的网络管理与时钟同步机制。
这些能力与高分辨环境感知所需的核心硬件基础高度重合。因此，相比单独建设专用感知基础设施，在现有通信平台上“叠加感知能力”通常具有更低的边际成本。
\paragraph{4) Network and data sharing}
ISAC 的成本优势不仅体现在“发射与接收设备共享”，还体现在“信息链路共享”。传统独立感知系统往往需要专门的数据回传、目标融合与控制接口；而在 ISAC 中，感知结果可以直接通过通信网络上传至边缘服务器或控制中心，通信链路本身也可承载传感数据、目标状态和协同控制信息，从而减少额外的网络基础设施投入。

\subsubsection{The limitations of hardware sharing and the engineering costs}
\label{subsubsec:ch1-isac-core value-hardware-limitations}

尽管硬件共享能够显著降低总体投入，但这种共享并非没有代价。根本原因在于：通信与感知在性能目标上并不完全一致，甚至在某些硬件指标上存在明显张力。因此，ISAC 的成本优势应理解为\textbf{系统级总成本降低}，而非“每一个单独器件都更便宜”。
\begin{itemize}
\item \textbf{动态范围（dynamic range）要求更高：}  
雷达式感知中，目标回波功率通常远低于发射信号，且不同目标、不同距离、不同散射截面积对应的回波强度差异很大。接收机需要在强泄漏、强近距回波与弱远距回波并存的情况下保持稳定工作，因此常要求更高的动态范围。相比之下，通信接收机虽然也需适应不同用户的接收功率差异，但其设计中心通常不是极弱回波检测。若 ISAC 平台需要同时满足这两类需求，则 ADC 位宽、自动增益控制和前端线性设计都可能需要增强，从而带来额外成本\cite{Wei2023Integrated}。
\item \textbf{收发隔离与自干扰抑制更困难：}  
在单站 ISAC，尤其是同时发射与接收的工作模式下，发射泄漏、自干扰和平台杂波会严重掩盖目标回波。为了可靠感知，系统往往需要较强的天线隔离、模拟域抵消和数字域干扰消除机制。这些措施会增加天线布局、射频链路和校准电路的复杂度\cite{Sturm2011WaveformDesign, Hassanien2016signaling}。
\item \textbf{时钟稳定度与相位噪声约束更严格：}  
高精度距离、多普勒和微多普勒估计通常对载波稳定性、采样时钟抖动和相位噪声更加敏感。通信系统虽然也受相位噪声影响，但很多情况下可通过同步、导频和均衡进行补偿；而感知中的精细速度估计、人体活动识别和高分辨参数估计往往更依赖稳定本振。因此，ISAC 共享时钟设计通常需要向感知侧的精度需求靠拢\cite{Sen2013YouAreWhat}。
\item \textbf{功率放大器在线性度与效率之间面临折中：}  
通信系统，特别是采用 OFDM 等高峰均功率比（PAPR）波形时，对功率放大器线性度要求较高；而传统雷达更强调高效率、大能量发射，有时偏好恒模或低 PAPR 波形。ISAC 波形和硬件必须同时兼顾可通信性与可感知性，这使得功率放大器、数字预失真和波形设计之间的协同优化更为关键\cite{Sturm2011WaveformDesign, Hassanien2016signaling}。
\item \textbf{标定与一致性维护成本不可忽视：}  
共享阵列和共享射频链路虽然减少了硬件数量，但也使系统更依赖精确校准。阵元幅相误差、通道不一致、温漂和机械偏移不仅影响通信波束成形，也会直接损害感知角度估计和成像质量。因此，大规模 ISAC 部署往往需要更加系统化的在线标定与健康监测机制，这会转化为额外的工程复杂度和软件维护成本。
\item \textbf{并非所有传感器都能被“完全替代”：}  
需要指出的是，ISAC 所强调的硬件共享主要适用于\textbf{基于无线电信号的感知}，如距离、速度、角度、占用状态和粗粒度环境感知等。对于某些对语义细节、纹理、颜色或高精度三维轮廓要求极高的任务，摄像头和激光雷达仍然具有不可替代的优势。因此，在自动驾驶、机器人或工业检测中，ISAC 更可能成为多传感器系统中的关键组成部分，而不是在所有场景下完全替代其他感知硬件\cite{Wei2023Integrated}。
\end{itemize}

综上，ISAC 的硬件共享并不是“零代价共享”，而是通过\textbf{适度增加统一平台的设计复杂度}，换取\textbf{网络级、系统级和全生命周期成本的降低}。从工程经济学角度看，只要共享带来的 CAPEX/OPEX 节省大于满足双功能需求所增加的器件、校准和算法成本，ISAC 就具有明确的硬件经济性优势。后续章节将进一步讨论硬件非理想性、收发机架构以及面向实际部署的工程实现问题。

\subsection{Deployment efficiency advantage}
\label{subsec:ch1-isac-core value-deployment}
\subsubsection{Why does the future network require high deployment efficiency}
\label{subsubsec:ch1-isac-core value-deployment-why}

未来网络的目标已不再局限于高速连接，而是向通信、感知、计算与控制深度融合的综合信息基础设施演进。面向 6G、智能交通、低空经济、工业互联网和智慧城市等场景，网络不仅要具备更强性能，还必须具备更高的部署效率、更好的扩展性和更灵活的升级能力。

一方面，未来应用场景覆盖更广、变化更快、需求更多样。例如，低空经济需要网络同时支持无人机通信与空域感知；车路协同要求道路基础设施兼具信息交互与环境感知能力；工业互联网则要求网络同时提供高可靠连接、低时延协同和现场状态感知\cite{Zhang2021ISAC6G, Feng2020JointRadarCommunication, Shafi20175G}。这类场景通常站点多、环境复杂、业务迭代快，因此网络若部署周期过长，将直接制约新业务落地与规模化推广。

另一方面，传统通信与感知系统独立建设的模式难以适应未来网络的敏捷部署需求。该模式往往需要分别建设通信与感知设备，并独立完成选址、供电、回传、频谱协调和运维管理，容易造成资源重复投入、系统协同复杂以及建设周期拉长。对于大型活动保障、灾后应急通信、临时交通管制等动态场景，这种模式尤其缺乏灵活性。

同时，网络演进正朝着软件化、虚拟化和平台化方向发展。5G 以后，越来越多的网络能力可以通过软件升级、云化部署和开放接口实现扩展\cite{Andrews2014, Letaief2019Roadmap}。在这一背景下，ISAC 提出以通信基础设施为载体，将感知能力内生为网络基础能力之一，使网络能够通过软件升级、参数重构和少量硬件增强快速获得感知能力，而无需额外建设大规模独立系统。

因此，未来网络需要高部署效率，根本原因在于应用扩张快、业务变化频繁，而传统独立建网模式成本高、周期长。ISAC 所倡导的通信与感知融合部署，为提升网络部署效率、服务敏捷性和可运营性提供了重要技术路径\cite{Liu2022ISAC, IMT2030Framework,ITU2030}。

\subsubsection{ISAC's mechanism for enhancing deployment efficiency}
\label{subsubsec:ch1-isac-core value-deployment-mechanism}

ISAC 提升部署效率的核心机制可以概括为：\textbf{依托现有通信基础设施、网络能力和运维体系，以较低增量成本快速加载感知功能}。这一机制不仅减少了硬件重复建设，也缩短了站点规划、安装调试、网络接入和业务开通的时间链条。
\paragraph{1) Reuse of existing communication infrastructure}
当前蜂窝网络已经形成了覆盖广泛、站点密集、体系成熟的基础设施。这些基础设施所具备的能力与无线感知所需的许多基础条件高度重合。因此，若能够通过软件升级、波形重构和有限硬件扩展，使基站在提供通信服务的同时具备环境感知能力，那么感知网络的建设就不必再从“站址申请—设备采购—安装施工—接入调试”的完整流程重新开始，而可以在已有网络上进行快速叠加。
从工程视角看，这种方式至少带来四方面的部署收益：
\begin{itemize}
    \item \textbf{减少新增站址需求：} 感知能力附着于既有站点，无需大量新建独立感知点位；
    \item \textbf{减少土建与配套周期：} 既有供电、散热、机柜、铁塔和回传资源可直接复用；
    \item \textbf{减少系统集成工作量：} 感知功能可接入既有网管、编排和边缘平台；
    \item \textbf{降低审批与协调复杂度：} 减少新增设备所引发的站址审批、施工协调和运维接入流程。
\end{itemize}
\paragraph{2) Unified planning for newly deployed nodes}
对于新建网络或新扩展区域，ISAC 还能够通过“统一规划、统一建设、统一开通”的方式提升部署效率。若通信与感知由同一平台承载，则站点选址、覆盖规划、供电设计、回传接入、边缘处理和网络管理可以在同一工程流程内完成，而不必针对通信网络和感知网络分别进行规划和建设。  
这种统一规划的优势不仅体现在缩短建设周期，还体现在减少跨系统协调和后期改造难度。特别是在道路、园区、港口、机场、矿区等场景中，统一部署往往比“先建通信、后补感知”更具工程可行性。
\paragraph{3) Faster service rollout}
从业务开通角度看，ISAC 显著降低了新型感知业务上线的门槛。传统模式下，若某一区域需要新增无人机监管、交通态势监测或周界环境感知能力，通常需要重新规划专用传感节点、布设回传链路，并建立单独的数据处理与运维流程。而在 ISAC 模式下，运营者可以优先在既有通信节点上激活相关功能，通过软件更新、参数配置和少量硬件增强实现能力上线。因此，ISAC 使网络能力的新增更接近于“平台升级”而非“重新建网”。这一点对于需要快速试点、快速复制和快速扩容的新兴业务尤其重要。
\paragraph{4) Deployment efficiency from software-defined evolution}
ISAC 的另一个重要部署优势在于其与软件定义网络演进路径相契合。随着无线网络越来越依赖软件配置、云化基带、开放接口和智能编排，很多原本需要通过新增专用设备实现的功能，正在转变为通过软件重构与资源调度实现\cite{Andrews2014,Letaief2019Roadmap}。  
在这一趋势下，感知能力若能够以内生网络功能的形式提供，就意味着其上线过程可更加接近通信功能扩展的方式，即：按需启用、区域开通、策略配置和逐步升级。这使得 ISAC 在部署效率上不仅优于传统独立感知方案，也更符合未来网络云化与软硬件解耦的发展方向。

当然，上述优势通常建立在一个前提之上：通信平台本身必须具备一定的感知潜力，例如足够的带宽、阵列规模、时钟稳定度和计算资源。如果目标应用对感知精度、探测范围或目标类别识别能力提出极高要求，则仍可能需要增加专用模块甚至引入其他传感器协同。因此，ISAC 在部署效率上的优势，本质上是一种\textbf{“尽可能利用已有网络平台实现快速感知能力扩展”}的系统性优势，而不是在任何场景下都完全无需额外建设。

\subsubsection{From single-node efficiency to network-level efficiency}
\label{subsubsec:ch1-isac-core value-deployment-network}

ISAC 的部署优势不仅体现在单节点建设简化上，更重要的是能够在网络级形成规模效应与平滑演进能力。

\paragraph{1) Efficiency gain at the single-node level}
在单站层面，ISAC 可由一个节点同时承担通信与部分感知功能，从而减少设备数量、安装环节以及天线、供电、机柜和回传等重复配置。对于杆站空间紧张、设备密集的场景，这种融合部署方式可有效降低建设与维护压力。

\paragraph{2) Native support for multi-site collaboration}
在网络层面，通信系统原有的时间同步、频率同步、站间接口和资源协调机制，也可为多站协同感知提供支撑\cite{Zhang2021ISAC6G,Liu2022ISAC}。因此，与独立建设协同感知网络相比，ISAC 能复用既有基础设施，降低网络级部署复杂度。

\paragraph{3) Easier scaling from local trial to wide-area deployment}
ISAC 还支持从局部试点到广域部署的渐进式演进。运营者可先在热点区域或重点场景开通感知功能，验证效果后再逐步扩展。这种方式有利于分阶段投资、边部署边优化，并复用现有运维体系，降低升级风险。

\paragraph{4) From engineering efficiency to ecosystem efficiency}
进一步看，ISAC 的价值不仅在于工程效率，也在于平台化扩展潜力。依托统一的网络平台，设备厂商、运营商和行业用户可围绕统一接口、统一管理和统一数据流开展协同创新\cite{IMT2030Framework,Letaief2019Roadmap}，从而推动标准化、规模化部署。

\paragraph{5) A necessary caveat}
需要指出的是，ISAC 并不意味着所有感知任务都可由现有通信网络直接承担。其效果仍受网络拓扑、频段、带宽、站点布局和遮挡条件等因素影响；在高精度成像或高鲁棒性感知场景中，往往仍需与摄像头、雷达或激光雷达等协同部署\cite{Zhang2021ISAC6G, Wei2023Integrated}。

综上，ISAC 的部署效率优势体现为：单节点减少重复建设，网络层面复用既有协同机制，演进路径上支持渐进升级，产业层面促进平台化扩展。因此，ISAC 特别适合作为未来网络中快速引入感知能力的重要技术方向。

\subsection{Advantage in Business Closed-Loop Capability}
\label{subsec:ch1-isac-core value-closed loop}

不同于传统网络主要承担“信息传递”功能，通信感知一体化（ISAC）不仅支持数据传输，还能够实时感知外部环境、目标状态和业务对象，为上层应用提供直接、及时的状态信息，从而支撑“感知—传输—决策—执行—反馈”的完整业务闭环\cite{Liu2022ISAC, Zhang2021ISAC6G, ITU2030, IMT2030Framework}。  

因此，ISAC 推动网络由“连接工具”进一步演进为“业务闭环支撑平台”，使其不仅承载业务数据，还能够深度参与业务过程本身。对于 6G 面向的智能交通、低空经济、工业自动化、智慧园区和公共安全等场景，仅有高速低时延通信已难以满足高自治、高实时和高可靠需求，网络还需具备物理世界感知能力，并与计算、控制过程紧密协同\cite{Letaief2019Roadmap, Saad2020Vision6G, ITU2030}。  

随着业务模式从“人发起、人决策、人执行”逐步转向“设备自主运行、系统实时协同、网络参与控制”，业务闭环能力正成为未来网络由“提供连接”迈向“支撑智能业务执行”的关键标志。

\subsubsection{The limitations of traditional communication systems}
\label{subsubsec:ch1-isac-core value-closed loop-communication}

传统通信系统的核心目标是实现用户、终端、设备与平台之间的信息可靠传递，其设计通常围绕带宽、时延、可靠性、覆盖、连接密度和频谱效率等指标展开\cite{Goldsmith2005,Andrews2014, Shafi20175G}。这一体系在移动互联网、物联网和宽带接入中已十分成熟，但若从“业务闭环”角度看，传统通信系统仍存在明显局限：\textbf{它擅长传递信息，却通常不直接具备对外部物理环境的认知能力}。

具体来说，传统通信网络虽然能够承载摄像头、雷达、工业传感器和车载终端的数据，但网络本身通常不直接获取环境状态，也难以主动识别目标位置、运动轨迹、行为变化和事件演化等信息。也就是说，传统通信系统能够回答“信息如何高效可靠地传输”，却难以直接回答“环境中发生了什么、目标状态如何变化以及系统应如何响应”。

从架构上看，环境感知与状态判断通常依赖网络外部的专用设备和独立平台。例如，车路协同依赖摄像头、毫米波雷达和激光雷达，工业自动化依赖专用传感器网络，低空经济依赖独立监视与探测设施。通信网络主要负责数据回传和命令下发，而环境理解、状态估计和决策控制多在网络外部完成。

这种“感知在外、通信在中、决策在后”的分离式架构，在未来高实时、高自治和大规模协同场景中会暴露出以下问题。

\paragraph{1) Long processing chain and additional latency}
传统业务流程通常需要经历“外部感知—网络传输—平台分析—生成指令—网络下发执行”的长链条。由于感知、传输、计算与控制分布在多个异构系统中，容易引入额外时延和交互开销\cite{Letaief2019Roadmap, Saad2020Vision6G}，从而影响自动驾驶、协作机器人和应急处置等实时业务的闭环响应速度。

\paragraph{2) Complex interfaces and weak cross-system coordination}
通信、感知和控制系统通常由不同设备、协议和平台构成，接口标准、数据格式和同步机制往往不统一，增加了系统集成与跨域协同难度。即使各子系统性能较强，整体业务闭环仍可能受限于接口瓶颈、同步误差和信息丢失。

\paragraph{3) Lack of global situational awareness at the network side}
传统网络本身不直接参与环境感知，因此通常只能掌握流量、连接和资源状态，而难以获取目标状态、场景状态和任务状态。这样一来，网络优化往往局限于吞吐量、时延和负载均衡等通信指标，难以实现面向任务的联合优化。

\paragraph{4) Difficult to support autonomous and closed-loop control}
未来许多场景要求网络直接参与“感知—分析—决策—反馈”的闭环过程\cite{Saad2020Vision6G, Liu2022ISAC}。如果网络自身缺乏感知能力，就必须依赖外部系统提供结果，从而降低闭环反应速度、稳定性和自治程度。因此，传统通信系统更适合支撑开环式信息服务，而不完全适合强实时自治闭环业务。

\paragraph{5) Limited capability for continuous optimization}
业务闭环不仅要求一次性感知与控制，还要求系统根据执行结果持续调整资源配置和决策策略。由于传统通信网络缺乏对物理世界状态的直接观测能力，通常难以独立完成“感知结果—业务结果—网络调度策略”之间的持续反馈优化\cite{IMT2030Framework, Letaief2019Roadmap}。

综上，传统通信系统的局限并不在于通信能力不足，而在于其角色主要被定义为\textbf{信息管道}，而非\textbf{环境认知与业务闭环的一部分}。随着未来网络向通感算控融合演进，这种定位已难以满足高实时、高自治和强协同应用的需求\cite{Liu2022ISAC, ITU2030, Saad2020Vision6G}。因此，ISAC 的意义不仅在于提升频谱效率或减少硬件重复建设，更在于使网络能够在同一体系中融合连接能力与环境认知能力，从而为构建真正的业务闭环网络奠定基础。

\subsubsection{ISAC forms a closed loop of ``perception $-$ transmission $-$ decision $-$ feedback''}
\label{subsubsec:ch1-isac-core value-closed loop-isac}

ISAC 的重要系统价值之一，在于推动网络形成“感知—传输—决策—执行—反馈”的业务闭环，使网络从单纯的信息传输平台演进为具备环境认知、任务支撑与动态优化能力的智能基础设施\cite{Liu2022ISAC, Zhang2021ISAC6G, Saad2020Vision6G, ITU2030}。  
不同于传统模式中感知、通信、决策和控制相互分离，ISAC 将无线感知能力内生到通信网络中，显著缩短状态获取、信息共享、任务决策与执行反馈之间的链路，从而提升系统实时性、自治性和鲁棒性。

从系统运行角度看，该闭环通常包括五个环节：\textbf{感知（perception）、传输（transmission）、决策（decision）、执行（execution）和反馈（feedback）}。

\begin{enumerate}[label=(\arabic*)]
\item \textbf{感知：获取环境与目标状态}

ISAC 利用一体化信号、共享频谱与基础设施，在通信过程中同步获取目标和环境信息，如距离、速度、角度、位置、轨迹及场景变化等\cite{Sturm2011WaveformDesign, Hassanien2016signaling, Liu2022ISAC}。其关键在于使网络不仅“知道谁在通信”，还能够“感知外部环境处于何种状态”。

\item \textbf{传输：共享状态并下发控制信息}

感知结果需要及时传送给相关节点、边缘平台或控制中心，而决策生成的控制命令也需通过通信链路可靠下发。因此，通信在闭环中同时承担状态同步、协同支撑和命令分发三类功能，构成整个闭环的信息流动骨架\cite{ITU2030, IMT2030Framework}。

\item \textbf{决策：基于感知结果进行调度与控制}

在获得环境状态后，网络侧或应用侧可进一步执行资源调度、风险评估、路径规划和控制优化等任务\cite{Letaief2019Roadmap, Saad2020Vision6G}。在 ISAC 中，决策既可发生在应用层，也可发生在网络层，例如根据目标位置和环境变化自适应调整波束、资源分配或感知策略。

\item \textbf{执行：将决策转化为实际动作}

决策生成后，需要落实为物理动作或网络动作，例如车辆减速、无人机改航、机器人避障，以及波束重配置、链路切换和资源重调度等\cite{Saad2020Vision6G}。执行环节体现了闭环能力最终作用于物理世界或网络系统。

\item \textbf{反馈：持续感知并修正策略}

执行后，ISAC 可通过持续感知再次观测目标与环境状态，并将更新结果反馈给系统，用于评估效果、修正偏差和优化后续决策\cite{Zhang2021ISAC6G}。由此形成动态循环：
\[
\text{感知} \rightarrow \text{传输} \rightarrow \text{决策} \rightarrow \text{执行} \rightarrow \text{反馈}.
\]
相比开环模式，该机制能够更有效应对环境不确定性和外部扰动，提升系统稳定性与自适应能力\cite{Saad2020Vision6G}。
\end{enumerate}

因此，ISAC 的闭环能力并非通信与感知功能的简单叠加，而是通过二者深度融合，贯通状态获取、信息共享、智能决策、任务执行与反馈修正的全过程\cite{Liu2022ISAC, Feng2020JointRadarCommunication}。对于智能交通、低空管控、工业控制和公共安全等高实时、高自治场景，这种闭环支撑能力正是 ISAC 区别于传统通信系统的核心系统价值之一。

\subsubsection{The significance of business loop capability for typical applications}
\label{subsubsec:ch1-isac-core value-closed loop-applications}

ISAC 的重要价值，不仅在于网络增加了感知能力，更在于其能够支撑“感知—通信—决策—执行—反馈”的业务闭环，使网络从单纯的信息传输平台演进为深度参与任务运行的智能基础设施\cite{Liu2022ISAC, Zhang2021ISAC6G, Saad2020Vision6G, ITU2030}。  
在智能交通、低空经济、工业自动化、公共安全等场景中，业务过程通常具有持续演化、动态调整和反复反馈的特征，因此，闭环能力往往比单纯的高速通信能力更能决定系统的实际运行效果。

从系统层面看，ISAC 的业务闭环能力主要体现在三个方面：  
\textbf{第一，提升实时响应能力。} 感知、传输与控制在统一网络中联动完成，可缩短从状态变化到控制响应的链路；  
\textbf{第二，增强自治运行能力。} 网络可直接获取环境状态并参与决策，减少对人工或外部独立系统的依赖；  
\textbf{第三，降低跨系统协同成本。} 通信、感知与控制在更统一的平台上实现，有助于减少接口割裂、信息孤岛和多系统协调开销。

这一意义在典型应用中表现尤为突出。  
\textbf{在车路协同场景中}，ISAC使网络不仅能够传递车路信息，还能够直接参与道路目标检测、轨迹估计和态势更新，从而支撑协同驾驶、风险预警和交通调度闭环，提升安全性与通行效率\cite{Kumari2018IEEE80211adRadar,Ma2026integrated,Feng2020JointRadarCommunication}。  
\textbf{在低空经济场景中}，ISAC可将目标探测、状态回传、轨迹预测、冲突规避和控制下发有机联结，支撑空域感知、调度与控制一体化，提高低空运行的安全性与管理效率\cite{Jiang2025integrated,Meng2023uav}。  
\textbf{在工业自动化场景中}，ISAC能够利用统一无线基础设施同时完成设备状态感知与工业数据传输，支撑生产监测、控制执行和反馈优化闭环，从而提升柔性制造和协作控制的实时性与鲁棒性\cite{Akyildiz2020Industrial6G,Kaushik2024integrated,Sturm2011WaveformDesign}。  
\textbf{在公共安全场景中}，ISAC有助于实现异常发现、信息汇聚、协同决策和行动反馈的快速闭环，从而提升态势感知、风险预警和联动处置能力\cite{Ma2022WiFiSensingSurvey,Liu2022ISAC,Zhang2021ISAC6G}。  
\textbf{在智能家居与智慧空间场景中}，ISAC使网络能够结合用户行为和环境状态持续调整服务策略，推动无线网络从“连接设备”走向“理解用户与环境”的服务闭环\cite{Ma2022WiFiSensingSurvey,Zhang2020device}。

总体来看，业务闭环能力的根本意义在于：它使网络能够更深入地嵌入业务流程，推动系统实现从“看见环境”到“理解环境”、从“传递信息”到“驱动决策”、再到“执行动作”与“持续优化”的能力升级。就方法论而言，这也是 ISAC 区别于传统“通信系统+感知系统简单拼接”的关键所在：其目标不是功能并置，而是通过统一的信号、资源与系统设计，构建面向任务执行的闭环智能基础设施。

\subsubsection{The business closed-loop capability is the key for ISAC to achieve its 6G inherent capabilities}
\label{subsubsec:ch1-isac-core value-closed loop-6g}

从网络演进的角度看，\textbf{业务闭环能力}是ISAC从“功能集成”走向“6G内生能力”的关键。只有当感知结果能够持续进入业务流程，并参与网络运行、资源调度、任务执行和反馈优化时，通信与感知的融合才具有真正的体系化价值 \cite{Liu2022ISAC,Zhang2021ISAC6G,Saad2020Vision6G}。

首先，\textbf{业务闭环使感知从附属测量功能演进为网络服务能力的重要组成部分}。传统通信系统以信息传输为核心，测距、定位和目标探测等通常只是附加能力 \cite{Sturm2011WaveformDesign}。而在ISAC中，感知结果可直接支撑链路建立、波束管理、资源调度、移动性管理和应用控制 \cite{Kaushik2024toward}。因此，感知不再是可选功能，而是网络实现高可靠、低时延和环境自适应服务的基础能力。对6G而言，这意味着网络不仅要传输比特，还要理解场景、感知状态并支撑行动 \cite{Saad2020Vision6G,Strinati20216G}。

其次，\textbf{业务闭环能力是ISAC迈向网络内生智能与系统自治的基础}。6G网络的发展目标不仅是更高的速率和连接密度，更在于提升对环境、用户和任务的理解能力，实现自适应、自优化和自管理 \cite{Saad2020Vision6G,Strinati20216G}。ISAC的价值也因此不只在于频谱共享或硬件复用，更在于它能够将物理世界的感知结果纳入网络控制闭环。例如，网络可依据目标位置和运动状态调整波束与接入策略，依据环境变化优化链路配置和切换决策 \cite{Liu2022ISAC,Zhang2021ISAC6G}。若缺乏闭环机制，感知信息就难以从“辅助观测”转化为网络自治所需的决策输入。

再次，\textbf{业务闭环能力为“ISAC+”及“通感算控一体化”提供了接口基础}。未来6G将推动通信、感知、计算与控制的深度协同，以支撑自动驾驶、低空协同、数字孪生等复杂任务型业务。若ISAC仅停留在波形复用、频谱共用和硬件共享层面，其作用仍主要是局部优化 \cite{Sturm2011WaveformDesign}。而具备业务闭环后，感知结果可以进一步驱动边缘计算、任务理解和控制执行，并通过持续反馈形成完整循环 \cite{Shi2016EdgeComputing,Chen2024integrated}。因此，业务闭环不仅提升了ISAC的应用深度，也为通感算控一体化提供了方法基础。

最后，\textbf{业务闭环能力决定了ISAC能否成为6G网络中的基础能力与核心服务能力}。所谓“原生智能”或“原生感知”，并不只是增加新的功能模块，而是要求这些能力嵌入网络运行机制，服务于业务执行和系统优化 \cite{Saad2020Vision6G,Strinati20216G}。对于ISAC而言，只有贯通感知、传输、决策与反馈的连续循环，才能从技术概念上升为网络原生能力 \cite{Liu2022ISAC,Kaushik2024toward}。

因此，\textbf{业务闭环能力既体现了ISAC区别于传统通信和分离式感知系统的本质优势，也决定了其能否真正融入6G网络架构，成为未来任务型业务的基础支撑能力}。这也表明，未来ISAC研究不能只关注通信性能与感知性能的并列提升，更应关注感知信息如何进入网络决策链条、参与业务控制，并在闭环运行中持续创造系统级价值。

\subsection{Reflective Questions}
\label{subsec:ch1.2-reflect-questions}

本节思考题围绕 ISAC 的系统级价值展开，重点考察读者对“系统能力重构”“系统级收益重构”以及频谱效率、硬件成本、部署效率和业务闭环能力等核心价值的理解。建议读者在完成本节学习后独立作答，其中部分题目可作为课堂讨论或课程作业选题。

\paragraph{一、概念理解类}
\begin{enumerate}
    \item 为什么说 ISAC 不是“可有可无的功能叠加”，而是一种系统能力的重构？请结合“独立通信系统”“独立感知系统”和“融合系统”的差异进行说明。

    \item 本节指出，ISAC 的核心价值不在于某一项单点性能指标的提升，而在于系统级收益的重构。请解释“系统级收益重构”的含义，并说明它与“某一项指标更优”之间的本质区别。

    \item 为什么说传统通信系统的设计目标主要面向信息传输，而非环境状态获取，因此在业务层面上常表现为“对物理世界无感”？为什么这种局限在 5G 以前可能并不突出，而在 6G 时代会变得越来越明显？请结合具体应用场景加以说明。
\end{enumerate}

\paragraph{二、对比分析类}
\begin{enumerate}
    \setcounter{enumi}{3}
    \item 请从以下四个方面，对比两种部署方案的差异，并分析哪一种更适合面向 6G 的大规模新型场景：
    \begin{enumerate}
        \item 频谱占用方式；
        \item 硬件建设成本；
        \item 部署流程与实施复杂度；
        \item 业务支撑能力。
    \end{enumerate}
    两种方案分别为：
    \begin{enumerate}
        \item 通信系统与感知系统独立部署；
        \item 采用 ISAC 的一体化部署。
    \end{enumerate}

    \item 本节从频谱效率、硬件成本、部署效率和业务闭环能力四个方面分析了 ISAC 的核心价值。请说明这四类价值是否彼此独立；如果不是，请分析它们之间可能存在的耦合关系与相互促进机制。

    \item 频谱效率提升并不意味着“任何场景下 ISAC 都一定优于独立系统”。请结合以下因素，分析 ISAC 频谱增益成立的适用条件及其边界：
    \begin{enumerate}
        \item 双功能之间的资源冲突；
        \item 互扰与自干扰；
        \item 目标分辨率要求；
        \item 通信链路质量要求。
    \end{enumerate}
\end{enumerate}

\paragraph{三、深度思考与论述类}
\begin{enumerate}
    \setcounter{enumi}{6}
    \item 假设某城市计划建设车路协同基础设施，有两种备选方案：

    方案 A：分别建设通信基站和路侧雷达；

    方案 B：在通信基站上升级 ISAC 能力。

    请从 CAPEX、OPEX、部署周期、维护复杂度和后续业务扩展能力五个角度，对两种方案进行比较，并给出你的建议。

    \item 本节提到，ISAC 通过共享天线阵列、射频链路和基带处理平台等硬件资源，能够降低系统总成本。请思考：尽管通信与感知在动态范围、线性度、时钟精度和收发隔离等方面的需求并不完全一致，为什么硬件共享在很多场景下仍可能是总体上更经济的方案？请从“系统总成本”而非“单模块最优性能”的角度进行论述。

    \item 请围绕“部署效率”这一价值，回答以下问题：
    \begin{enumerate}
        \item 为什么说 ISAC 特别适合需要快速、广域和规模化部署的低空经济、工业互联网和公共安全等新兴场景？
        \item 如果这些场景完全依赖独立感知基础设施，在部署上可能会遇到哪些现实障碍？
    \end{enumerate}

    \item 本节提出，ISAC 的最大价值之一在于形成“感知—传输—决策—反馈”的业务闭环。请选择以下任一场景进行分析：车路协同、低空无人机管理、智能工厂或公共安全监测。要求说明：
    \begin{enumerate}
        \item 感知环节获取什么信息；
        \item 通信环节传输什么内容；
        \item 由谁完成决策；
        \item 反馈如何作用于下一轮系统行为。
    \end{enumerate}

    \item 有一种观点认为：“只要在通信系统外接一些雷达或传感器，也能实现环境感知，因此没有必要发展 ISAC。”请结合本节内容，对这一观点进行评述。要求至少从以下三个方面展开：
    \begin{enumerate}
        \item 系统成本；
        \item 部署效率；
        \item 业务闭环能力。
    \end{enumerate}

    \item 请论述为什么说业务闭环能力是 ISAC 走向 6G 内生能力的关键，而不仅仅是一个附加优势。提示：可从网络角色变化、智能化需求、数字孪生以及“通感算控”一体化等角度展开。

    \item 假设未来 6G 网络已普遍具备 ISAC 能力，那么运营商的角色将发生怎样的变化？其是否仍主要扮演“连接服务提供者”的角色，还是会进一步演进为“环境信息服务提供者”乃至“决策支撑平台”？请谈谈你的理解。
\end{enumerate}

\paragraph{四、开放探索类}
\begin{enumerate}
    \setcounter{enumi}{13}
    \item 如果你是一名系统设计者，需要为某一类应用场景评估 ISAC 的实际价值，你会从哪些维度建立评估指标体系？请至少给出 5 个评价维度，并说明每个维度分别对应 ISAC 的哪一类核心价值。构建指标体系时，应尽量兼顾技术性能、部署成本与业务价值。

    \item 请思考：在未来一段时间内，ISAC 最可能率先在哪些行业实现规模化落地？是车联网、低空经济、工业自动化、智慧城市，还是其他场景？请结合“频谱效率、硬件成本、部署效率、业务闭环能力”四个维度给出理由。
